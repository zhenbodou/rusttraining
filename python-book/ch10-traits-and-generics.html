Rust for Python Programmers

Trait vs 鸭子类型

你将学到： Trait 作为显式契约（vs Python 鸭子类型），Protocol（PEP 544）≈ Trait， 使用 where 子句的泛型类型约束，trait 对象（dyn Trait）vs 静态分派，以及常见 std trait。

难度： 🟡 中级

这是 Rust 类型系统对 Python 开发者真正闪耀的地方。Python 的 "鸭子类型"说："如果它像鸭子一样走路并且像鸭子一样叫，它就是鸭子。" Rust 的 trait 说："我会在编译时准确告诉你我需要哪些鸭子行为。"

Python 鸭子类型

# Python —— 鸭子类型：有任何正确方法的东西都可以工作
def total_area(shapes):
    """适用于任何有 .area() 方法的东西。"""
    return sum(shape.area() for shape in shapes)

class Circle:
    def __init__(self, radius): self.radius = radius
    def area(self): return 3.14159 * self.radius ** 2

class Rectangle:
    def __init__(self, w, h): self.w, self.h = w, h
    def area(self): return self.w * self.h

# 运行时工作 —— 不需要继承！
shapes = [Circle(5), Rectangle(3, 4)]
print(total_area(shapes))  # 90.54

# 但如果某物没有 .area() 会怎样？
class Dog:
    def bark(self): return "Woof!"

total_area([Dog()])  # 💥 AttributeError: 'Dog' has no attribute 'area'
# 错误在运行时发生，不是在定义时

Rust Trait —— 显式鸭子类型

#![allow(unused)]
fn main() {
// Rust —— trait 使"鸭子"契约显式
trait HasArea {
    fn area(&self) -> f64;      # 任何实现这个 trait 的类型都有 .area()
}

struct Circle { radius: f64 }
struct Rectangle { width: f64, height: f64 }

impl HasArea for Circle {
    fn area(&self) -> f64 {
        std::f64::consts::PI * self.radius * self.radius
    }
}

impl HasArea for Rectangle {
    fn area(&self) -> f64 {
        self.width * self.height
    }
}

trait 约束是显式的 —— 编译器在编译时检查
fn total_area(shapes: &[&dyn HasArea]) -> f64 {
    shapes.iter().map(|s| s.area()).sum()
}

使用它：
let shapes: Vec<&dyn HasArea> = vec![&Circle { radius: 5.0 }, &Rectangle { width: 3.0, height: 4.0 }];
println!("{}", total_area(&shapes));  # 90.54

struct Dog;
total_area(&[&Dog {}]);  # ❌ 编译错误：Dog 没有实现 HasArea
}

关键见解：Python 的鸭子类型将错误推迟到运行时。Rust 的 trait 在 编译时捕获它们。相同的灵活性，更早的错误检测。

Protocol（PEP 544）vs Trait

Python 3.8 引入了 Protocol（PEP 544）用于结构子类型 —— 它是 最接近 Rust trait 的 Python 概念。

Python Protocol

# Python —— Protocol（结构类型，如 Rust trait）
from typing import Protocol, runtime_checkable

@runtime_checkable
class Printable(Protocol):
    def to_string(self) -> str: ...

class User:
    def __init__(self, name: str):
        self.name = name
    def to_string(self) -> str:
        return f"User({self.name})"

class Product:
    def __init__(self, name: str, price: float):
        self.name = name
        self.price = price
    def to_string(self) -> str:
        return f"Product({self.name}, ${self.price:.2f})"

def print_all(items: list[Printable]) -> None:
    for item in items:
        print(item.to_string())

# 工作因为 User 和 Product 都有 to_string()
print_all([User("Alice"), Product("Widget", 9.99)])

# 但是：mypy 检查这个，Python 运行时不强制执行
# print_all([42])  # mypy 警告，但 Python 运行并崩溃

Rust Trait（等价，但强制执行！）

#![allow(unused)]
fn main() {
// Rust —— trait 在编译时强制执行
trait Printable {
    fn to_string(&self) -> String;
}

struct User { name: String }
struct Product { name: String, price: f64 }

impl Printable for User {
    fn to_string(&self) -> String {
        format!("User({})", self.name)
    }
}

impl Printable for Product {
    fn to_string(&self) -> String {
        format!("Product({}, ${:.2})", self.name, self.price)
    }
}

fn print_all(items: &[&dyn Printable]) {
    for item in items {
        println!("{}", item.to_string());
    }
}

print_all(&[&42i32]);  # ❌ 编译错误：i32 没有实现 Printable
}

泛型约束

Python 泛型

# Python —— TypeVar 用于泛型函数
from typing import TypeVar, Sequence

T = TypeVar('T')

def first(items: Sequence[T]) -> T | None:
    return items[0] if items else None

# 有界 TypeVar
from typing import SupportsFloat
T = TypeVar('T', bound=SupportsFloat)

def average(items: Sequence[T]) -> float:
    return sum(float(x) for x in items) / len(items)

Rust 带 Trait 约束的泛型

#![allow(unused)]
fn main() {
// Rust —— 带 trait 约束的泛型
fn first<T>(items: &[T]) -> Option<&T> {
    items.first()
}

带 trait 约束 —— "T 必须实现这些 trait"
fn average<T>(items: &[T]) -> f64
where
    T: Into<f64> + Copy,   # T 必须转换为 f64 且可复制
{
    let sum: f64 = items.iter().map(|&x| x.into()).sum();
    sum / items.len() as f64
}

多个约束 —— "T 必须实现 Display 和 Debug 和 Clone"
fn log_and_clone<T: std::fmt::Display + std::fmt::Debug + Clone>(item: &T) -> T {
    println!("Display: {}", item);
    println!("Debug: {:?}", item);
    item.clone()
}

简单情况的 impl Trait 简写
fn print_it(item: &impl std::fmt::Display) {
    println!("{}", item);
}
}

关联类型

Rust trait 可以定义关联类型 —— 每个实现者填写的类型占位符。Python 没有等价物：

#![allow(unused)]
fn main() {
Iterator 定义关联类型 'Item'
trait Iterator {
    type Item;
    fn next(&mut self) -> Option<Self::Item>;
}

struct Countdown { remaining: u32 }

impl Iterator for Countdown {
    type Item = u32;  # 这个迭代器产生 u32 值
    fn next(&mut self) -> Option<u32> {
        if self.remaining > 0 {
            self.remaining -= 1;
            Some(self.remaining)
        } else {
            None
        }
    }
}
}

在 Python 中，__iter__ / __next__ 返回 Any —— 无法声明"这个迭代器产生 int"并强制执行（带 Iterator[int] 的类型提示仅是建议性的）。

运算符重载：__add__ → impl Add

Python 使用魔术方法（__add__、__mul__）。Rust 使用 trait 实现 —— 相同想法，但在编译时类型检查：

# Python
class Vec2:
    def __init__(self, x, y):
        self.x, self.y = x, y
    def __add__(self, other):
        return Vec2(self.x + other.x, self.y + other.y)  # 'other' 上无类型检查

#![allow(unused)]
fn main() {
use std::ops::Add;

#[derive(Debug, Clone, Copy)]
struct Vec2 { x: f64, y: f64 }

impl Add for Vec2 {
    type Output = Vec2;  # 关联类型：+ 返回什么？
    fn add(self, rhs: Vec2) -> Vec2 {
        Vec2 { x: self.x + rhs.x, y: self.y + rhs.y }
    }
}

let a = Vec2 { x: 1.0, y: 2.0 };
let b = Vec2 { x: 3.0, y: 4.0 };
let c = a + b;  # 类型安全：只允许 Vec2 + Vec2
}

关键差异：Python 的 __add__ 在运行时接受任何 other（你手动检查类型或得到 TypeError）。Rust 的 Add trait 在编译时强制执行操作数类型 —— Vec2 + i32 是编译错误，除非你显式 impl Add<i32> for Vec2。

练习

🏋️ 练习：泛型 Summary Trait（点击展开）

挑战：定义一个 trait Summary，方法为 fn summarize(&self) -> String。为两个 struct 实现它：Article { title: String, body: String } 和 Tweet { username: String, content: String }。然后编写一个函数 fn notify(item: &impl Summary) 打印摘要。

🔑 解答 
trait Summary {
    fn summarize(&self) -> String;
}

struct Article { title: String, body: String }
struct Tweet { username: String, content: String }

impl Summary for Article {
    fn summarize(&self) -> String {
        format!("{} — {}...", self.title, &self.body[..20.min(self.body.len())])
    }
}

impl Summary for Tweet {
    fn summarize(&self) -> String {
        format!("@{}: {}", self.username, self.content)
    }
}

fn notify(item: &impl Summary) {
    println!("📢 {}", item.summarize());
}

fn main() {
    let article = Article {
        title: "Rust is great".into(),
        body: "Here is why Rust beats Python for systems...".into(),
    };
    let tweet = Tweet {
        username: "rustacean".into(),
        content: "Just shipped my first crate!".into(),
    };
    notify(&article);
    notify(&tweet);
}

关键要点：&impl Summary 是 Python 的 Protocol 带 summarize 方法的 Rust 等价物。但 Rust 在编译时检查它 —— 传递未实现 Summary 的类型是编译错误，不是运行时 AttributeError。
