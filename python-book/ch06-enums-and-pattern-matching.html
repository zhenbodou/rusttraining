Rust for Python Programmers

代数数据类型 vs Union 类型

你将学到： 带数据的 Rust enum vs Python Union 类型，穷尽 match vs match/case， Option<T> 作为 None 的编译时替代品，以及 guard 模式。

难度： 🟡 中级

Python 3.10 引入了 match 语句和类型联合。Rust 的 enum 更进一步 —— 每个变体可以携带不同的数据，编译器确保你处理每种情况。

Python Union 类型和 Match

# Python 3.10+ —— 结构模式匹配
from typing import Union
from dataclasses import dataclass

@dataclass
class Circle:
    radius: float

@dataclass
class Rectangle:
    width: float
    height: float

@dataclass
class Triangle:
    base: float
    height: float

Shape = Union[Circle, Rectangle, Triangle]  # 类型别名

def area(shape: Shape) -> float:
    match shape:
        case Circle(radius=r):
            return 3.14159 * r * r
        case Rectangle(width=w, height=h):
            return w * h
        case Triangle(base=b, height=h):
            return 0.5 * b * h
        # 如果漏掉情况，没有编译器警告！
        # 添加新形状？grep 代码库并希望找到所有 match 块。

Rust Enum —— 携带数据的变体

#![allow(unused)]
fn main() {
// Rust —— enum 变体携带数据，编译器强制执行穷尽匹配
enum Shape {
    Circle(f64),                # Circle 携带半径
    Rectangle(f64, f64),        # Rectangle 携带宽、高
    Triangle { base: f64, height: f64 }, # 也可以使用命名字段
}

fn area(shape: &Shape) -> f64 {
    match shape {
        Shape::Circle(r) => std::f64::consts::PI * r * r,
        Shape::Rectangle(w, h) => w * h,
        Shape::Triangle { base, height } => 0.5 * base * height,
        ❌ 如果你添加 Shape::Pentagon 并忘记在这里处理，
           编译器拒绝构建。无需 grep。
    }
}
}

关键见解：Rust 的 match 是穷尽的 —— 编译器验证你处理 每个变体。向 enum 添加新变体，编译器会准确告诉你哪些 match 块需要更新。Python 的 match 没有这种保证。

Enum 替换多个 Python 模式

# Python —— Rust enum 替换的几种模式：

# 1. 字符串常量
STATUS_PENDING = "pending"
STATUS_ACTIVE = "active"
STATUS_CLOSED = "closed"

# 2. Python Enum（无数据）
from enum import Enum
class Status(Enum):
    PENDING = "pending"
    ACTIVE = "active"
    CLOSED = "closed"

# 3. 标记联合（class + 类型字段）
class Message:
    def __init__(self, kind, **data):
        self.kind = kind
        self.data = data
# Message(kind="text", content="hello")
# Message(kind="image", url="...", width=100)

enum Status {
    Pending,

穷尽模式匹配

Python 的 match —— 不穷尽

# Python —— 通配符情况是可选的，无编译器帮助
def describe(value):
    match value:
        case 0:
            return "zero"
        case 1:
            return "one"
        # 如果忘记默认值，Python 静默返回 None。
        # 无警告，无错误。

describe(42)  # 返回 None —— 静默 bug

Rust 的 match —— 编译器强制执行

#![allow(unused)]
fn main() {
// Rust —— 必须处理每种可能情况
fn describe(value: i32) -> &'static str {
    match value {
        0 => "zero",
        1 => "one",
        ❌ 编译错误：非穷尽模式：`i32::MIN..=-1_i32`
           和 `2_i32..=i32::MAX` 未覆盖
        _ => "other",   # _ = 通配符（开放类型需要）
    }
}

对于 enum，不需要通配符 —— 编译器知道所有变体：
enum Color { Red, Green, Blue }

fn color_hex(c: Color) -> &'static str {
    match c {
        Color::Red => "#ff0000",
        Color::Green => "#00ff00",
        Color::Blue => "#0000ff",
        不需要 _ —— 所有变体已覆盖
        稍后添加 Color::Yellow → 此处的编译错误
    }
}
}

模式匹配功能

#![allow(unused)]
fn main() {
多个值（如 Python 的 case 1 | 2 | 3：）
match value {
    1 | 2 | 3 => println!("small"),
    4..=9 => println!("medium"),    # 范围模式
    _ => println!("large"),
}

Guard（如 Python 的 case x if x > 0：）
match temperature {
    t if t > 100 => println!("boiling"),
    t if t < 0 => println!("freezing"),
    t => println!("normal: {t}°"),
}

嵌套 destructuring
let point = (3, (4, 5));
match point {
    (0, _) => println!("on y-axis"),
    (_, (0, _)) => println!("y=0"),
    (x, (y, z)) => println!("x={x}, y={y}, z={z}"),
}
}

Option 实现 None 安全

Option<T> 是 Python 开发者最重要的 Rust enum。它用类型安全的替代品替换 None。

Python None

# Python —— None 是可以出现在任何地方的值
def find_user(user_id: int) -> dict | None:
    users = {1: {"name": "Alice"}}
    return users.get(user_id)

user = find_user(999)
# user 是 None —— 但没有什么强制你检查！
print(user["name"])  # 💥 运行时 TypeError

Rust Option

#![allow(unused)]
fn main() {
// Rust —— Option<T> 强制你处理 None 情况
fn find_user(user_id: i64) -> Option<User> {
    let users = HashMap::from([(1, User { name: "Alice".into() })]);
    users.get(&user_id).cloned()
}

let user = find_user(999);
user 是 Option<User> —— 你不能在不处理 None 的情况下使用它

方法 1：match
match find_user(999) {
    Some(user) => println!("Found: {}", user.name),
    None => println!("Not found"),
}

方法 2：if let（如 Python 的 if (x := expr) is not None）
if let Some(user) = find_user(1) {
    println!("Found: {}", user.name);
}

方法 3：unwrap_or
let name = find_user(999)
    .map(|u| u.name)
    .unwrap_or_else(|| "Unknown".to_string());

方法 4：? 运算符（在返回 Option 的函数中）
fn get_user_name(id: i64) -> Option<String> {
    let user = find_user(id)?;     # 如果未找到则提前返回 None
    Some(user.name)
}
}

练习

🏋️ 练习：Shape 面积计算器（点击展开）

挑战：定义一个 enum Shape，变体为 Circle(f64)（半径）、Rectangle(f64, f64)（宽、高）和 Triangle(f64, f64)（底、高）。使用 match 实现方法 fn area(&self) -> f64。创建每种一个并打印面积。

🔑 解答 
use std::f64::consts::PI;

enum Shape {
    Circle(f64),
    Rectangle(f64, f64),
    Triangle(f64, f64),
}

impl Shape {
    fn area(&self) -> f64 {
        match self {
            Shape::Circle(r) => PI * r * r,
            Shape::Rectangle(w, h) => w * h,
            Shape::Triangle(b, h) => 0.5 * b * h,
        }
    }
}

fn main() {
    let shapes = [
        Shape::Circle(5.0),
        Shape::Rectangle(4.0, 6.0),
        Shape::Triangle(3.0, 8.0),
    ];
    for shape in &shapes {
        println!("Area: {:.2}", shape.area());
    }
}

关键要点：Rust enum 替换 Python 的 Union[Circle, Rectangle, Triangle] + isinstance() 检查。编译器确保你处理每个变体 —— 添加新形状而不更新 area() 是编译错误。
