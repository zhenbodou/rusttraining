Rust for Python Programmers

Rust 闭包 vs Python Lambda

你将学到： 多行闭包（不仅是一表达式 lambda），Fn/FnMut/FnOnce 捕获语义， 迭代器链 vs 列表推导式，map/filter/fold，以及 macro_rules! 基础。

难度： 🟡 中级

Python 闭包和 Lambda

# Python —— lambda 是单表达式匿名函数
double = lambda x: x * 2
result = double(5)  # 10

# 完整闭包从外部作用域捕获变量：
def make_adder(n):
    def adder(x):
        return x + n    # 从外部作用域捕获 `n`
    return adder

add_5 = make_adder(5)
print(add_5(10))  # 15

# 高阶函数：
numbers = [1, 2, 3, 4, 5]
doubled = list(map(lambda x: x * 2, numbers))
evens = list(filter(lambda x: x % 2 == 0, numbers))

Rust 闭包

// Rust —— 闭包使用 |args| body 语法
let double = |x: i32| x * 2;
let result = double(5);  # 10

闭包从外部作用域捕获变量：

    move |x| x + n    # `move` 将 `n` 的所有权转移到闭包中

}

闭包捕获 —— Rust 如何不同

# Python —— 闭包通过引用捕获（后期绑定！）
funcs = [lambda: i for i in range(3)]
print([f() for f in funcs])  # [2, 2, 2] —— 惊喜！都捕获同一个 `i`

# 用默认参数技巧修复：
funcs = [lambda i=i: i for i in range(3)]
print([f() for f in funcs])  # [0, 1, 2]

#![allow(unused)]
fn main() {
// Rust —— 闭包正确捕获（无后期绑定问题！）
let funcs: Vec<Box<dyn Fn() -> i32>> = (0..3)
    .map(|i| Box::new(move || i) as Box<dyn Fn() -> i32>)
    .collect();

let results: Vec<i32> = funcs.iter().map(|f| f()).collect();
println!("{:?}", results);  # [0, 1, 2] —— 正确！

`move` 为每个闭包捕获 `i` 的副本 —— 无后期绑定惊喜。
}

三个闭包 Trait

#![allow(unused)]
fn main() {
// Rust 闭包实现这些 trait 中的一个或多个：

Fn —— 可以多次调用，不突变捕获（最常见）
fn apply(f: impl Fn(i32) -> i32, x: i32) -> i32 { f(x) }

FnMut —— 可以多次调用，可能突变捕获
fn apply_mut(mut f: impl FnMut(i32) -> i32, x: i32) -> i32 { f(x) }

FnOnce —— 只能调用一次（消耗捕获）
fn apply_once(f: impl FnOnce() -> String) -> String { f() }

Python 没有等价物 —— 闭包始终像 Fn。
在 Rust 中，编译器自动确定使用哪个 trait。
}

迭代器 vs 生成器

Python 生成器

# Python —— 使用 yield 的生成器
def fibonacci():
    a, b = 0, 1
    while True:
        yield a
        a, b = b, a + b

# 惰性 —— 值按需计算
fib = fibonacci()
first_10 = [next(fib) for _ in range(10)]

# 生成器表达式 —— 如惰性列表推导式
squares = (x ** 2 for x in range(1000000))  # 无内存分配
first_5 = [next(squares) for _ in range(5)]

Rust 迭代器

#![allow(unused)]
fn main() {
// Rust —— Iterator trait（概念相似，语法不同）
struct Fibonacci {
    a: u64,
    b: u64,
}

impl Fibonacci {
    fn new() -> Self {
        Fibonacci { a: 0, b: 1 }
    }
}

impl Iterator for Fibonacci {
    type Item = u64;

    fn next(&mut self) -> Option<Self::Item> {
        let current = self.a;
        self.a = self.b;
        self.b = current + self.b;
        Some(current)
    }
}

惰性 —— 值按需计算（如 Python 生成器）
let first_10: Vec<u64> = Fibonacci::new().take(10).collect();

迭代器链 —— 如生成器表达式
let squares: Vec<u64> = (0..1_000_000u64).map(|x| x * x).take(5).collect();
}

推导式 vs 迭代器链

本节将 Python 的推导式语法映射到 Rust 的迭代器链。

列表推导式 → map/filter/collect

# Python 推导式：
squares = [x ** 2 for x in range(10)]
evens = [x for x in range(20) if x % 2 == 0]
names = [user.name for user in users if user.active]
pairs = [(x, y) for x in range(3) for y in range(3)]
flat = [item for sublist in nested for item in sublist]

flowchart LR
    A["源\n[1,2,3,4,5]"] -->|.iter\(\)| B["迭代器"]
    B -->|.filter\(\|x\| x%2==0\)| C["[2, 4]"]
    C -->|.map\(\|x\| x*x\)| D["[4, 16]"]
    D -->|.collect\(\)| E["Vec&lt;i32&gt;\n[4, 16]"]
    style A fill:#ffeeba
    style E fill:#d4edda

关键见解：Rust 迭代器是惰性的 —— 直到 .collect() 才发生什么。Python 的生成器类似工作，但列表推导式急切求值。

#![allow(unused)]
fn main() {
// Rust 迭代器链：
let squares: Vec<i32> = (0..10).map(|x| x * x).collect();
let evens: Vec<i32> = (0..20).filter(|x| x % 2 == 0).collect();
let names: Vec<&str> = users.iter()
    .filter(|u| u.active)
    .map(|u| u.name.as_str())
    .collect();
let pairs: Vec<(i32, i32)> = (0..3)
    .flat_map(|x| (0..3).map(move |y| (x, y)))
    .collect();
let flat: Vec<i32> = nested.iter()
    .flat_map(|sublist| sublist.iter().copied())
    .collect();
}

Dict 推导式 → collect 到 HashMap

# Python
word_lengths = {word: len(word) for word in words}
inverted = {v: k for k, v in mapping.items()}

#![allow(unused)]
fn main() {
// Rust
let word_lengths: HashMap<&str, usize> = words.iter()
    .map(|w| (*w, w.len()))
    .collect();
let inverted: HashMap<&V, &K> = mapping.iter()
    .map(|(k, v)| (v, k))
    .collect();
}

Set 推导式 → collect 到 HashSet

# Python
unique_lengths = {len(word) for word in words}

#![allow(unused)]
fn main() {
// Rust
let unique_lengths: HashSet<usize> = words.iter()
    .map(|w| w.len())
    .collect();
}

关键差异

Python 迭代器：                     Rust 迭代器：
─────────────────                     ──────────────
- 默认惰性（生成器）               - 默认惰性（所有迭代器链）
- yield 创建生成器                - impl Iterator { fn next() }
- StopIteration 结束              - None 结束
- 可以消耗一次                    - 可以消耗一次
- 无类型安全                      - 完全类型安全
- 稍慢（解释器）                  - 零成本（编译掉）

为什么宏存在于 Rust 中

Python 没有宏系统 —— 它使用装饰器、metaclass 和运行时 内省进行元编程。Rust 使用宏进行编译时代码生成。

Python 元编程 vs Rust 宏

# Python —— 装饰器和 metaclass 用于元编程
from dataclasses import dataclass
from functools import wraps

@dataclass              # 在导入时生成 __init__、__repr__、__eq__
class Point:
    x: float
    y: float

# 自定义装饰器
def log_calls(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        print(f"Calling {func.__name__}")
        return func(*args, **kwargs)
    return wrapper

@log_calls
def process(data):
    return data.upper()

#![allow(unused)]
fn main() {
// Rust —— 派生宏和声明宏用于代码生成
#[derive(Debug, Clone, PartialEq)]  # 在编译时生成 Debug、Clone、PartialEq 实现
struct Point {
    x: f64,
    y: f64,
}

声明宏（如模板）
macro_rules! log_call {
    ($func_name:expr, $body:expr) => {
        println!("Calling {}", $func_name);
        $body
    };
}

fn process(data: &str) -> String {
    log_call!("process", data.to_uppercase())
}
}

#[derive(...)] 是 Python 的 @dataclass 装饰器的 Rust 等价物

Python：
@dataclass(frozen=True, order=True)
class Student:
    name: str
    grade: int

Rust：
#[derive(Debug, Clone, PartialEq, Eq, PartialOrd, Ord, Hash)]
struct Student {
    name: String,
    grade: i32,
}

常见派生宏：
Debug         → {:?} 格式化（如 __repr__）
Clone         → .clone() 深复制
Copy          → 隐式复制（仅用于简单类型）
PartialEq, Eq → == 比较（如 __eq__）
PartialOrd, Ord → <, >, 排序（如 __lt__ 等）
Hash          → 可用作 HashMap 键（如 __hash__）
Default       → MyType::default()（如无参数 __init__）

crate 提供的派生宏：
Serialize, Deserialize (serde) → JSON/YAML/TOML 序列化
                                 （如 Python 的 json.dumps/loads 但类型安全）
}

练习

🏋️ 练习：派生和自定义 Debug（点击展开）

挑战：创建一个 User struct，字段为 name: String、email: String 和 password_hash: String。派生 Clone 和 PartialEq，但手动实现 Debug，使其打印名称和电子邮件但隐藏密码（显示 "***" 而不是）。

🔑 解答 
use std::fmt;

#[derive(Clone, PartialEq)]
struct User {
    name: String,
    email: String,
    password_hash: String,
}

impl fmt::Debug for User {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        f.debug_struct("User")
            .field("name", &self.name)
            .field("email", &self.email)
            .field("password_hash", &"***")
            .finish()
    }
}

fn main() {
    let user = User {
        name: "Alice".into(),
        email: "alice@example.com".into(),
        password_hash: "a1b2c3d4e5f6".into(),
    };
    println!("{user:?}");
    输出：User { name: "Alice", email: "alice@example.com", password_hash: "***" }
}

关键要点：与 Python 的 __repr__ 不同，Rust 让你免费派生 Debug —— 但你可以为敏感字段覆盖它。这比 Python 更安全，在 Python 中 print(user) 可能意外泄露秘密。
