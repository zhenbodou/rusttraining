Rust for Python Programmers

Rust 中的类型转换

你将学到： From 和 Into trait 用于零成本类型转换，TryFrom 用于可能失败的转换， 如何 impl From<A> for B 自动生成 Into，以及 string 转换模式。

难度： 🟡 中级

Python 使用构造函数调用处理类型转换（int("42")、str(42)、 float("3.14")）。Rust 使用 From 和 Into trait 进行类型安全转换。

Python 类型转换

# Python —— 显式构造函数用于转换
x = int("42")           # str → int（可以抛出 ValueError）
s = str(42)             # int → str
f = float("3.14")       # str → float
lst = list((1, 2, 3))   # tuple → list

# 通过 __init__ 或类方法自定义转换
class Celsius:
    def __init__(self, temp: float):
        self.temp = temp

    @classmethod
    def from_fahrenheit(cls, f: float) -> "Celsius":
        return cls((f - 32.0) * 5.0 / 9.0)

c = Celsius.from_fahrenheit(212.0)  # 100.0°C

Rust From/Into

#![allow(unused)]
fn main() {
// Rust —— From trait 定义转换
// 实现 From<T> 自动给你 Into<U>！

struct Celsius(f64);
struct Fahrenheit(f64);

impl From<Fahrenheit> for Celsius {
    fn from(f: Fahrenheit) -> Self {
        Celsius((f.0 - 32.0) * 5.0 / 9.0)
    }
}

现在两者都工作：
let c1 = Celsius::from(Fahrenheit(212.0));    # 显式 From
let c2: Celsius = Fahrenheit(212.0).into();   # Into（自动派生）

String 转换：
let s: String = String::from("hello");         # &str → String
let s: String = "hello".to_string();           # 相同
let s: String = "hello".into();                # 也工作（From 已实现）

let num: i64 = 42i32.into();                   # i32 → i64（无损，所以存在 From）
let small: i32 = 42i64.into();              # ❌ i64 → i32 可能丢失数据 —— 无 From

对于可能失败的转换，使用 TryFrom：
let n: Result<i32, _> = "42".parse();          # str → i32（可能失败）
let n: i32 = "42".parse().unwrap();            # 如果不是数字则 panic
let n: i32 = "42".parse()?;                    # 用 ? 传播错误
}

经验法则：始终实现 From，从不直接实现 Into。实现 From<A> for B 免费给你 Into<B> for A。

何时使用 From/Into

为你的类型实现 From<T> 以实现符合人体工程学的 API 设计：

#[derive(Debug)]
struct UserId(i64);

impl From<i64> for UserId {
    fn from(id: i64) -> Self {
        UserId(id)
    }
}

现在函数可以接受任何可转换为 UserId 的东西：
fn find_user(id: impl Into<UserId>) -> Option<String> {
    let user_id = id.into();
    ... 查找逻辑
    Some(format!("User #{:?}", user_id))
}

find_user(42i64);              # ✅ i64 自动转换为 UserId
find_user(UserId(42));         # ✅ UserId 保持原样
}

TryFrom —— 可能失败的转换

不是所有转换都能成功。Python 抛出异常；Rust 使用 TryFrom 返回 Result：

# Python —— 可能失败的转换抛出异常
try:
    port = int("not_a_number")   # ValueError
except ValueError as e:
    print(f"Invalid: {e}")

# __init__ 中的自定义验证
class Port:
    def __init__(self, value: int):
        if not (1 <= value <= 65535):
            raise ValueError(f"Invalid port: {value}")
        self.value = value

try:
    p = Port(99999)  # 运行时 ValueError
except ValueError:
    pass

#![allow(unused)]
fn main() {
use std::num::ParseIntError;

TryFrom 用于内置类型
let n: Result<i32, ParseIntError> = "42".try_into();   # Ok(42)
let n: Result<i32, ParseIntError> = "bad".try_into();  # Err(...)

自定义 TryFrom 用于验证
#[derive(Debug)]
struct Port(u16);

#[derive(Debug)]
enum PortError {
    OutOfRange(u16),
    Zero,
}

impl TryFrom<u16> for Port {
    type Error = PortError;

    fn try_from(value: u16) -> Result<Self, Self::Error> {
        match value {
            0 => Err(PortError::Zero),
            1..=65535 => Ok(Port(value)),
            注意：u16 最大是 65535，所以这覆盖所有情况
        }
    }
}

impl std::fmt::Display for PortError {
    fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> fmt::Result {
        match self {
            PortError::Zero => write!(f, "port cannot be zero"),
            PortError::OutOfRange(v) => write!(f, "port {v} out of range"),
        }
    }
}

使用：
let p: Result<Port, _> = 8080u16.try_into();   # Ok(Port(8080))
let p: Result<Port, _> = 0u16.try_into();       # Err(PortError::Zero)
}

Python → Rust 心智模型：TryFrom = __init__ 验证且可能失败。但不像抛出异常，它返回 Result —— 所以调用者必须处理错误情况。

String 转换模式

字符串是 Python 开发者转换混淆的最常见来源：

#![allow(unused)]
fn main() {
String → &str（借用，免费）
let s = String::from("hello");
let r: &str = &s;              # 自动 Deref 强制
let r: &str = s.as_str();     # 显式

&str → String（分配，花费内存）
let r: &str = "hello";
let s1 = String::from(r);     # From trait
let s2 = r.to_string();       # ToString trait（通过 Display）
let s3: String = r.into();    # Into trait

Number → String
let s = 42.to_string();       # "42" —— 如 Python 的 str(42)
let s = format!("{:.2}", 3.14); # "3.14" —— 如 Python 的 f"{3.14:.2f}"

String → Number
let n: i32 = "42".parse().unwrap();       # 如 Python 的 int("42")
let f: f64 = "3.14".parse().unwrap();     # 如 Python 的 float("3.14")

自定义类型 → String（实现 Display）
use std::fmt;

struct Point { x: f64, y: f64 }

impl fmt::Display for Point {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        write!(f, "({}, {})", self.x, self.y)
    }
}

let p = Point { x: 1.0, y: 2.0 };
println!("{p}");                # (1, 2) —— 如 Python 的 __str__
let s = p.to_string();         # 也工作！Display 免费给你 ToString。
}

转换链和错误处理

真实代码通常链式多个转换。比较方法：

# Python —— 转换链带 try/except
def parse_config(raw: str) -> tuple[str, int]:
    try:
        host, port_str = raw.split(":")
        port = int(port_str)
        if not (1 <= port <= 65535):
            raise ValueError(f"Bad port: {port}")
        return (host, port)
    except (ValueError, AttributeError) as e:
        raise ConfigError(f"Invalid config: {e}") from e

fn parse_config(raw: &str) -> Result<(String, u16), String> {
    let (host, port_str) = raw
        .split_once(':')
        .ok_or_else(|| "missing ':' separator".to_string())?;

    let port: u16 = port_str
        .parse()
        .map_err(|e| format!("invalid port: {e}"))?;

    if port == 0 {
        return Err("port cannot be zero".to_string());
    }

    Ok((host.to_string(), port))
}

fn main() {
    match parse_config("localhost:8080") {
        Ok((host, port)) => println!("Connecting to {host}:{port}"),
        Err(e) => eprintln!("Config error: {e}"),
    }
}

关键见解：每个 ? 都是可见的退出点。在 Python 中，try 内的任何行都可能抛出 —— 在 Rust 中，只有以 ? 结尾的行可能失败。

🏋️ 练习：温度转换库（点击展开）

挑战：构建一个迷你温度转换库：

  1. 定义 Celsius(f64)、Fahrenheit(f64) 和 Kelvin(f64) struct
  2. 实现 From<Celsius> for Fahrenheit 和 From<Celsius> for Kelvin
  3. 实现 TryFrom<f64> for Kelvin，拒绝低于绝对零度的值（-273.15°C = 0K）
  4. 为所有三种类型实现 Display（例如 "100.00°C"）
🔑 解答 
use std::fmt;

struct Celsius(f64);
struct Fahrenheit(f64);
struct Kelvin(f64);

impl From<Celsius> for Fahrenheit {
    fn from(c: Celsius) -> Self {
        Fahrenheit(c.0 * 9.0 / 5.0 + 32.0)
    }
}

impl From<Celsius> for Kelvin {
    fn from(c: Celsius) -> Self {
        Kelvin(c.0 + 273.15)
    }
}

#[derive(Debug)]
struct BelowAbsoluteZero;

impl fmt::Display for BelowAbsoluteZero {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        write!(f, "temperature below absolute zero")
    }
}

impl TryFrom<f64> for Kelvin {
    type Error = BelowAbsoluteZero;

    fn try_from(value: f64) -> Result<Self, Self::Error> {
        if value < 0.0 {
            Err(BelowAbsoluteZero)
        } else {
            Ok(Kelvin(value))
        }
    }
}

impl fmt::Display for Celsius    { fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result { write!(f, "{:.2}°C", self.0) } }
impl fmt::Display for Fahrenheit { fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result { write!(f, "{:.2}°F", self.0) } }
impl fmt::Display for Kelvin     { fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result { write!(f, "{:.2}K",  self.0) } }

fn main() {
    let boiling = Celsius(100.0);
    let f: Fahrenheit = Celsius(100.0).into();
    let k: Kelvin = Celsius(100.0).into();
    println!("{boiling} = {f} = {k}");

    match Kelvin::try_from(-10.0) {
        Ok(k) => println!("{k}"),
        Err(e) => println!("Error: {e}"),
    }
}

关键要点：From 处理不可失败的转换（Celsius→Fahrenheit 始终工作）。TryFrom 处理可能失败的转换（负 Kelvin 不可能）。Python 在 __init__ 中混淆两者 —— Rust 在类型系统中使区别显式。
