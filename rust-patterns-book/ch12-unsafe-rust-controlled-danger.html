五种 Unsafe 超能力

unsafe 解锁编译器无法验证的五种操作：

#![allow(unused)]
fn main() {
// 安全：每个操作在下面内联解释。
unsafe {
    // 1. 解引用原始指针
    let ptr: *const i32 = &42;
    let value = *ptr; // 可能是悬垂/空指针

    // 2. 调用 unsafe 函数
    let layout = std::alloc::Layout::new::<u64>();
    let mem = std::alloc::alloc(layout);

    // 3. 访问可变静态变量
    static mut COUNTER: u32 = 0;
    COUNTER += 1; // 如果多个线程访问则数据竞争

    // 4. 实现 unsafe trait
    // unsafe impl Send for MyType {}

    // 5. 访问 union 字段
    // union IntOrFloat { i: i32, f: f32 }
    // let u = IntOrFloat { i: 42 };
    // let f = u.f; // 重新解释位——可能是垃圾
}
}

关键原则：unsafe 不会关闭借用检查器或类型系统。 它只解锁这五种特定能力。所有其他 Rust 规则仍然适用。

编写可靠的抽象

unsafe 的目的是在 unsafe 操作周围构建安全抽象：

#![allow(unused)]
fn main() {
/// 固定容量栈分配缓冲区。
/// 所有公共方法都是安全的——unsafe 被封装。
pub struct StackBuf<T, const N: usize> {
    data: [std::mem::MaybeUninit<T>; N],
    len: usize,
}

impl<T, const N: usize> StackBuf<T, N> {
    pub fn new() -> Self {
        StackBuf {
            // 每个元素单独 MaybeUninit——无需 unsafe。
            // `const { ... }` 块（Rust 1.79+）让我们重复非 Copy
            // const 表达式 N 次。
            data: [const { std::mem::MaybeUninit::uninit() }; N],
            len: 0,
        }
    }

    pub fn push(&mut self, value: T) -> Result<(), T> {
        if self.len >= N {
            return Err(value); // 缓冲区满——返回值给调用者
        }
        // 安全：len < N，所以 data[len] 在边界内。
        // 我们将有效 T 写入 MaybeUninit 槽。
        self.data[self.len] = std::mem::MaybeUninit::new(value);
        self.len += 1;
        Ok(())
    }

    pub fn get(&self, index: usize) -> Option<&T> {
        if index < self.len {
            // 安全：index < len，且 data[0..len] 都初始化。
            Some(unsafe { self.data[index].assume_init_ref() })
        } else {
            None
        }
    }
}

impl<T, const N: usize> Drop for StackBuf<T, N> {
    fn drop(&mut self) {
        // 安全：data[0..len] 初始化——正确 drop 它们。
        for i in 0..self.len {
            unsafe { self.data[i].assume_init_drop(); }
        }
    }
}
}

可靠 unsafe 代码的三个规则：

  1. 文档化不变量 —— 每个 // SAFETY: 注释解释为什么操作有效
  2. 封装 —— unsafe 在安全 API 内；用户不能触发 UB
  3. 最小化 —— 只有最小可能的块是 unsafe

FFI 模式：从 Rust 调用 C

#![allow(unused)]
fn main() {
// 声明 C 函数签名：
extern "C" {
    fn strlen(s: *const std::ffi::c_char) -> usize;
    fn printf(format: *const std::ffi::c_char, ...) -> std::ffi::c_int;
}

// 安全包装器：
fn safe_strlen(s: &str) -> usize {
    let c_string = std::ffi::CString::new(s).expect("string contains null byte");
    // 安全：c_string 是有效 null 终止字符串，在调用期间存活。
    unsafe { strlen(c_string.as_ptr()) }
}

// 从 C 调用 Rust（导出函数）：
#[no_mangle]
pub extern "C" fn rust_add(a: i32, b: i32) -> i32 {
    a + b
}
}

何时在生产中使用 unsafe：

  • FFI 边界（调用 C/C++ 代码）
  • 性能关键内部循环（避免边界检查）
  • 构建原语（Vec、HashMap —— 这些内部使用 unsafe）
  • 如果可以避免，从不在应用逻辑中

自定义分配器 —— Arena 和 Slab 模式

在 C 中，你会为特定分配模式编写自定义 malloc() 替换—— 一次释放所有的 arena 分配器、固定大小对象的 slab 分配器、 或高吞吐量系统的池分配器。Rust 通过 GlobalAlloc trait 和分配器 crate 提供相同能力， 带生命周期作用域 arena 的额外好处，在编译时防止 use-after-free。

Arena 分配器——批量分配，批量释放

Arena 通过向前推进指针分配。单个项不能释放—— 整个 arena 一次释放。这对请求作用域或帧作用域分配完美：

#![allow(unused)]
fn main() {
use bumpalo::Bump;

fn process_sensor_frame(raw_data: &[u8]) {
    // 为此帧的分配创建 arena
    let arena = Bump::new();

    // 在 arena 中分配对象——每个约 2ns（只是指针推进）
    let header = arena.alloc(parse_header(raw_data));
    let readings: &mut [f32] = arena.alloc_slice_fill_default(header.sensor_count);

    for (i, chunk) in raw_data[header.payload_offset..].chunks(4).enumerate() {
        if i < readings.len() {
            readings[i] = f32::from_le_bytes(chunk.try_into().unwrap());
        }
    }

    // 使用 readings...
    let avg = readings.iter().sum::<f32>() / readings.len() as f32;
    println!("Frame avg: {avg:.2}");

    // `arena` 在这里 drop —— 所有分配一次在 O(1) 释放
    // 无每对象析构函数开销，无碎片
}
fn parse_header(_: &[u8]) -> Header { Header { sensor_count: 4, payload_offset: 8 } }
struct Header { sensor_count: usize, payload_offset: usize }
}

typed-arena —— 类型安全 Arena

当所有 arena 对象是相同类型时，typed-arena 提供更简单 API， 返回带 arena 生命周期的引用：

#![allow(unused)]
fn main() {
use typed_arena::Arena;

struct AstNode<'a> {
    value: i32,
    children: Vec<&'a AstNode<'a>>,
}

fn build_tree() {
    let arena: Arena<AstNode<'_>> = Arena::new();

    // 分配节点——返回绑定到 arena 生命周期的 &AstNode
    let root = arena.alloc(AstNode { value: 1, children: vec![] });
    let left = arena.alloc(AstNode { value: 2, children: vec![] });
    let right = arena.alloc(AstNode { value: 3, children: vec![] });

    // 构建树——所有引用只要 `arena` 存活就有效
    // （真正可变树需要内部可变性）

    println!("Root: {}, Left: {}, Right: {}", root.value, left.value, right.value);

    // `arena` 在这里 drop —— 所有节点一次释放
}
}

Slab 分配器——固定大小对象池

slab 分配器预分配固定大小槽的池。对象单独分配和返回， 但所有槽大小相同——消除碎片并启用 O(1) 分配/释放：

#![allow(unused)]
fn main() {
use slab::Slab;

struct Connection {
    id: u64,
    buffer: [u8; 1024],
    active: bool,
}

fn connection_pool_example() {
    // 为连接预分配 slab
    let mut connections: Slab<Connection> = Slab::with_capacity(256);

    // Insert 返回键（usize 索引）——O(1)
    let key1 = connections.insert(Connection {
        id: 1001,
        buffer: [0; 1024],
        active: true,
    });

    let key2 = connections.insert(Connection {
        id: 1002,
        buffer: [0; 1024],
        active: true,
    });

    // 通过键访问——O(1)
    if let Some(conn) = connections.get_mut(key1) {
        conn.buffer[0..5].copy_from_slice(b"hello");
    }

    // Remove 返回值——O(1)，槽为下次 insert 重用
    let removed = connections.remove(key2);
    assert_eq!(removed.id, 1002);

    // 下次 insert 重用释放的槽——无碎片
    let key3 = connections.insert(Connection {
        id: 1003,
        buffer: [0; 1024],
        active: true,
    });
    assert_eq!(key3, key2); // 相同槽重用！
}
}

实现最小 Arena（用于 no_std）

对于无法拉入 bumpalo 的裸机环境，这里是基于 unsafe 的最小 arena：

#![allow(unused)]
#![cfg_attr(not(test), no_std)]

fn main() {
use core::alloc::Layout;
use core::cell::{Cell, UnsafeCell};

/// 由固定大小字节数组支持的简单 bump 分配器。
/// 非线程安全——对多线程上下文使用每核心或带锁。
///
/// **重要**：像 `bumpalo`，这个 arena 在 arena drop 时不调用分配项的析构函数。
/// 带 `Drop` impl 的类型将泄漏其资源（文件句柄、套接字等）。
/// 只分配无有意义 `Drop` impl 的类型，或在 arena 之前手动 drop 它们。
pub struct FixedArena<const N: usize> {
    // UnsafeCell 在这里必需：我们通过 `&self` 变异 `buf`。
    // 无 UnsafeCell，将 &self.buf 转换为 *mut u8 将是 UB
    // （违反 Rust 的别名模型——共享引用意味着不可变）。
    buf: UnsafeCell<[u8; N]>,
    offset: Cell<usize>, // &self 分配的内部可变性
}

impl<const N: usize> FixedArena<N> {
    pub const fn new() -> Self {
        FixedArena {
            buf: UnsafeCell::new([0; N]),
            offset: Cell::new(0),
        }
    }

    /// 在 arena 中分配 `T`。如果空间不足返回 `None`。
    pub fn alloc<T>(&self, value: T) -> Option<&mut T> {
        let layout = Layout::new::<T>();
        let current = self.offset.get();

        // 向上对齐
        let aligned = (current + layout.align() - 1) & !(layout.align() - 1);
        let new_offset = aligned + layout.size();

        if new_offset > N {
            return None; // Arena 满
        }

        self.offset.set(new_offset);

        // 安全：
        // - `aligned` 在 `buf` 边界内（上面检查）
        // - 对齐正确（对齐到 T 的要求）
        // - 无别名：每个 alloc 返回唯一、不重叠区域
        // - UnsafeCell 授予通过 &self 变异的权限
        // - arena 比返回的引用存活更长（调用者必须确保）
        let ptr = unsafe {
            let base = (self.buf.get() as *mut u8).add(aligned);
            let typed = base as *mut T;
            typed.write(value);
            &mut *typed
        };

        Some(ptr)
    }

    /// 重置 arena —— 使所有先前分配无效。
    ///
    /// # 安全
    /// 调用者必须确保不存在对 arena 分配数据的引用。
    pub unsafe fn reset(&self) {
        self.offset.set(0);
    }

    pub fn used(&self) -> usize {
        self.offset.get()
    }

    pub fn remaining(&self) -> usize {
        N - self.offset.get()
    }
}
}

要点——Unsafe Rust

  • 文档化不变量（SAFETY: 注释），封装在安全 API 后，最小化 unsafe 作用域
  • [const { MaybeUninit::uninit() }; N]（Rust 1.79+）替换旧 assume_init 反模式
  • FFI 需要 extern "C"、#[repr(C)] 和仔细的空/生命周期处理
  • Arena 和 slab 分配器用通用灵活性交换分配速度

另见： 第 4 章 — PhantomData 了解方差和 drop-check 与 unsafe 代码的交互。第 8 章 — 智能指针 了解 Pin 和自引用类型。

练习：Unsafe 的安全包装器 ★★★（约 45 分钟）

编写 FixedVec<T, const N: usize> —— 固定容量、栈分配向量。 要求：

  • push(&mut self, value: T) -> Result<(), T> 满时返回 Err(value)
  • pop(&mut self) -> Option<T> 返回并移除最后一个元素
  • as_slice(&self) -> &[T] 借用初始化元素
  • 所有公共方法必须安全；所有 unsafe 必须用 SAFETY: 注释封装
  • Drop 必须清理初始化元素
🔑 解决方案 
use std::mem::MaybeUninit;

pub struct FixedVec<T, const N: usize> {
    data: [MaybeUninit<T>; N],
    len: usize,
}

impl<T, const N: usize> FixedVec<T, N> {
    pub fn new() -> Self {
        FixedVec {
            data: [const { MaybeUninit::uninit() }; N],
            len: 0,
        }
    }

    pub fn push(&mut self, value: T) -> Result<(), T> {
        if self.len >= N { return Err(value); }
        // 安全：len < N，所以 data[len] 在边界内。
        self.data[self.len] = MaybeUninit::new(value);
        self.len += 1;
        Ok(())
    }

    pub fn pop(&mut self) -> Option<T> {
        if self.len == 0 { return None; }
        self.len -= 1;
        // 安全：data[len] 初始化（len 在递减前 > 0）。
        Some(unsafe { self.data[self.len].assume_init_read() })
    }

    pub fn as_slice(&self) -> &[T] {
        // 安全：data[0..len] 都初始化，且 MaybeUninit<T>
        // 与 T 布局相同。
        unsafe { std::slice::from_raw_parts(self.data.as_ptr() as *const T, self.len) }
    }

    pub fn len(&self) -> usize { self.len }
    pub fn is_empty(&self) -> bool { self.len == 0 }
}

impl<T, const N: usize> Drop for FixedVec<T, N> {
    fn drop(&mut self) {
        // 安全：data[0..len] 初始化——drop 每个。
        for i in 0..self.len {
            unsafe { self.data[i].assume_init_drop(); }
        }
    }
}

fn main() {
    let mut v = FixedVec::<String, 4>::new();
    v.push("hello".into()).unwrap();
    v.push("world".into()).unwrap();
    assert_eq!(v.as_slice(), &["hello", "world"]);
    assert_eq!(v.pop(), Some("world".into()));
    assert_eq!(v.len(), 1);
}
