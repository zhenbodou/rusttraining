Mutex 选择指南

Mutex何时使用
std::sync::Mutex锁短暂持有，从不跨 .await
tokio::sync::Mutex锁必须跨 .await 持有
parking_lot::Mutex高竞争，无 .await，需要性能
tokio::sync::RwLock多读者，少写者，锁跨 .await

决策快速参考

需要并发？
├── I/O 绑定 → async/await
├── CPU 绑定 → rayon / std::thread
└── 混合 → spawn_blocking 用于 CPU 部分

选择运行时？
├── 服务器应用 → tokio
├── 库 → 运行时无关（futures crate）
├── 嵌入式 → embassy
└── 最小 → smol

需要并发 future？
├── 可以是 'static + Send → tokio::spawn
├── 可以是 'static + !Send → LocalSet
├── 不能是 'static → FuturesUnordered
└── 需要跟踪/中止 → JoinSet

常见错误消息和修复

错误原因修复
future is not Send在 .await 之间持有 !Send 类型作用域值以便在 .await 之前 drop，或使用 current_thread 运行时
borrowed value does not live long enough 在 spawn 中tokio::spawn 需要 'static使用 Arc、clone() 或 FuturesUnordered
the trait Future is not implemented for ()缺少 .await向异步调用添加 .await
cannot borrow as mutable 在 poll 中自引用借用正确使用 Pin<&mut Self>（见第 4 章）
程序静默挂起忘记调用 waker.wake()确保每个 Pending 路径注册并触发 waker
