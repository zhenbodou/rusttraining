Rust for C# Programmers

学习路径和下一步

你将学到： 结构化学习路线图（第 1-2 周，第 1-3 个月+），推荐书籍和资源， C# 开发者的常见陷阱（所有权混淆、与借用检查器斗争）， 以及带 tracing 的结构化可观察性 vs ILogger。

难度： 🟢 初学者

立即下一步（第 1-2 周）

  1. 设置你的环境

    • 通过 rustup.rs 安装 Rust
    • 配置 VS Code 带 rust-analyzer 扩展
    • 创建你的第一个 cargo new hello_world 项目

  2. 掌握基础

    • 用简单练习练习所有权
    • 编写带不同参数类型的函数（&str、String、&mut）
    • 实现基本结构体和方法

  3. 错误处理练习

    • 将 C# try-catch 代码转换为基于 Result 的模式
    • 练习 ? 运算符和 match 语句
    • 实现自定义错误类型

中级目标（第 1-2 个月）

  1. 集合和迭代器

    • 掌握 Vec<T>、HashMap<K,V> 和 HashSet<T>
    • 学习迭代器方法：map、filter、collect、fold
    • 练习 for 循环 vs 迭代器链

  2. Traits 和泛型

    • 实现常见 traits：Debug、Clone、PartialEq
    • 编写泛型函数和结构体
    • 理解 trait 边界和 where 子句

  3. 项目结构

    • 将代码组织到模块中
    • 理解 pub 可见性
    • 使用来自 crates.io 的外部 crates

高级主题（第 3 个月+）

  1. 并发

    • 了解 Send 和 Sync traits
    • 对基本并行性使用 std::thread
    • 探索 tokio 进行异步编程

  2. 内存管理

    • 理解 Rc<T> 和 Arc<T> 用于共享所有权
    • 学习何时对堆分配使用 Box<T>
    • 掌握复杂场景的生命周期

  3. 现实世界项目

    • 用 clap 构建 CLI 工具
    • 用 axum 或 warp 创建 Web API
    • 编写库并发布到 crates.io

C# 开发者的常见陷阱

所有权混淆

#![allow(unused)]
fn main() {
// 不要：尝试使用移动的值
fn wrong_way() {
    let s = String::from("hello");
    takes_ownership(s);
    // println!("{}", s); // 错误：s 被移动了
}

// 做：必要时使用引用或 clone
fn right_way() {
    let s = String::from("hello");
    borrows_string(&s);
    println!("{}", s); // 好：s 在这里仍然拥有
}

fn takes_ownership(s: String) { /* s 在这里被移动 */ }
fn borrows_string(s: &str) { /* s 在这里被借用 */ }
}

与借用检查器斗争

#![allow(unused)]
fn main() {
// 不要：多个可变引用
fn wrong_borrowing() {
    let mut v = vec![1, 2, 3];
    let r1 = &mut v;
    // let r2 = &mut v; // 错误：不能两次可变借用
}

// 做：限制可变借用的作用域
fn right_borrowing() {
    let mut v = vec![1, 2, 3];
    {
        let r1 = &mut v;
        r1.push(4);
    } // r1 在这里超出作用域

    let r2 = &mut v; // 好：无其他可变借用存在
    r2.push(5);
}
}

期望 Null 值

#![allow(unused)]
fn main() {
// 不要：期望 null 类似行为
fn no_null_in_rust() {
    // let s: String = null; // Rust 中无 null！
}

// 做：显式使用 Option<T>
fn use_option_instead() {
    let maybe_string: Option<String> = None;

    match maybe_string {
        Some(s) => println!("Got string: {}", s),
        None => println!("No string available"),
    }
}
}

结构化可观察性：tracing vs ILogger 和 Serilog

C# 开发者习惯于通过 ILogger、Serilog 或 NLog 进行结构化日志记录——其中日志消息携带类型化的键值属性。Rust 的 log crate 提供基本级别日志记录，但 tracing 是带 spans、异步感知和分布式跟踪支持的生产级结构化可观察性标准。

为什么 tracing 超过 log

特性log cratetracing crateC# 等价物
级别消息✅ info!()、error!()✅ info!()、error!()ILogger.LogInformation()
结构化字段❌ 仅字符串插值✅ 类型化键值字段Serilog Log.Information("{User}", user)
Spans（作用域上下文）❌✅ #[instrument]、span!()ILogger.BeginScope()
异步感知❌ 在 .await 上丢失上下文✅ Spans 跟随 .awaitActivity / DiagnosticSource
分布式跟踪❌✅ OpenTelemetry 集成System.Diagnostics.Activity
多种输出格式基本JSON、漂亮、紧凑、OTLPSerilog sinks

开始

# Cargo.toml
[dependencies]
tracing = "0.1"
tracing-subscriber = { version = "0.3", features = ["env-filter", "json"] }

基本用法：结构化日志记录

// C# Serilog
Log.Information("Processing order {OrderId} for {Customer}, total {Total:C}",
    orderId, customer.Name, order.Total);
// 输出：Processing order 12345 for Alice, total $99.95
// JSON:  {"OrderId": 12345, "Customer": "Alice", "Total": 99.95, ...}

#![allow(unused)]
fn main() {
use tracing::{info, warn, error, debug, instrument};

// 结构化字段——类型化，不是字符串插值
info!(order_id = 12345, customer = "Alice", total = 99.95,
      "Processing order");
// 输出：INFO Processing order order_id=12345 customer="Alice" total=99.95
// JSON:  {"order_id": 12345, "customer": "Alice", "total": 99.95, ...}

// 动态值
let order_id = 12345;
info!(order_id, "Order received");  // 字段名 = 变量名简写

// 条件字段
if let Some(promo) = promo_code {
    info!(order_id, promo_code = %promo, "Promo applied");
    //                        ^ % 意味着使用 Display 格式化
    //                        ? 将使用 Debug 格式化
}
}

Spans：异步代码的杀手级特性

Spans 是携带字段跨函数调用和 .await 点的作用域上下文——像 ILogger.BeginScope() 但异步安全。

// C# —— Activity / BeginScope
using var activity = new Activity("ProcessOrder").Start();
activity.SetTag("order_id", orderId);

using (_logger.BeginScope(new Dictionary<string, object> { ["OrderId"] = orderId }))
{
    _logger.LogInformation("Starting processing");
    await ProcessPaymentAsync();
    _logger.LogInformation("Payment complete");  // OrderId 仍然在作用域中
}

#![allow(unused)]
fn main() {
use tracing::{info, instrument, Instrument};

// #[instrument] 自动创建带函数参数作为字段的 span
#[instrument(skip(db), fields(customer_name))]
async fn process_order(order_id: u64, db: &Database) -> Result<(), AppError> {
    let order = db.get_order(order_id).await?;

    // 动态添加字段到当前 span
    tracing::Span::current().record("customer_name", &order.customer_name.as_str());

    info!("Starting processing");
    process_payment(&order).await?;        // span 上下文在 .await 上保留！
    info!(items = order.items.len(), "Payment complete");
    Ok(())
}
// 这个函数内的每个日志消息自动包括：
//   order_id=12345 customer_name="Alice"
// 甚至在嵌套异步调用中！

// 手动 span 创建（类似 BeginScope）
async fn batch_process(orders: Vec<u64>, db: &Database) {
    for order_id in orders {
        let span = tracing::info_span!("process_order", order_id);

        // .instrument(span) 将 span 附加到 future
        process_order(order_id, db)
            .instrument(span)
            .await
            .unwrap_or_else(|e| error!("Failed: {e}"));
    }
}
}

Subscriber 配置（类似 Serilog Sinks）

#![allow(unused)]
fn main() {
use tracing_subscriber::{fmt, EnvFilter, layer::SubscriberExt, util::SubscriberInitExt};

fn init_tracing() {
    // 开发：人类可读、彩色输出
    tracing_subscriber::registry()
        .with(EnvFilter::try_from_default_env()
            .unwrap_or_else(|_| "my_app=debug,tower_http=info".into()))
        .with(fmt::layer().pretty())  // 彩色、缩进 spans
        .init();
}

fn init_tracing_production() {
    // 生产：JSON 输出用于日志聚合（类似 Serilog JSON sink）
    tracing_subscriber::registry()
        .with(EnvFilter::new("my_app=info"))
        .with(fmt::layer().json())  // 结构化 JSON
        .init();
    // 输出：{"timestamp":"...","level":"INFO","fields":{"order_id":123},...}
}
}
# 通过环境变量控制日志级别（类似 Serilog MinimumLevel）
RUST_LOG=my_app=debug,hyper=warn cargo run
RUST_LOG=trace cargo run  // 一切

OpenTelemetry 集成

# 用于分布式跟踪（类似 System.Diagnostics + OTLP 导出器）
[dependencies]
tracing-opentelemetry = "0.22"
opentelemetry = "0.21"
opentelemetry-otlp = "0.14"

#![allow(unused)]
fn main() {
// 添加 OpenTelemetry 层与控制台输出一起
use tracing_opentelemetry::OpenTelemetryLayer;

fn init_otel() {
    let tracer = opentelemetry_otlp::new_pipeline()
        .tracing()
        .with_exporter(opentelemetry_otlp::new_exporter().tonic())
        .install_batch(opentelemetry_sdk::runtime::Tokio)
        .expect("Failed to create OTLP tracer");

    tracing_subscriber::registry()
        .with(OpenTelemetryLayer::new(tracer))  // 发送 spans 到 Jaeger/Tempo
        .with(fmt::layer())                      // 也打印到控制台
        .init();
}
// 现在 #[instrument] spans 自动成为分布式跟踪！
}
