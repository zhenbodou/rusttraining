Rust for C# Programmers

变量和可变性

你将学到： Rust 的变量声明和可变性模型与 C# 的 var/const 对比， 原始类型映射，关键的 String vs &str 区别，类型推断， 以及 Rust 如何处理与 C# 不同的转换和转换。

难度： 🟢 初学者

C# 变量声明

// C# - 变量默认可变
int count = 0;           // 可变
count = 5;               // ✅ 可以

// readonly 字段（仅限类级别，不能用于局部变量）
// readonly int maxSize = 100;  // 初始化后不可变

const int BUFFER_SIZE = 1024; // 编译时常量（可作为局部或字段）

Rust 变量声明

#![allow(unused)]
fn main() {
// Rust - 变量默认不可变
let count = 0;           // 默认不可变
// count = 5;            // ❌ 编译错误：不能两次赋值给不可变变量

let mut count = 0;       // 显式可变
count = 5;               // ✅ 可以

const BUFFER_SIZE: usize = 1024; // 编译时常量
}

C# 开发者的关键思维转变

#![allow(unused)]
fn main() {
// 将 'let' 想象为应用于所有变量的 C# readonly 字段语义
let name = "John";       // 类似 readonly 字段：一旦设置，不能更改
let mut age = 30;        // 类似：int age = 30;

// 变量遮蔽（Rust 独有）
let spaces = "   ";      // 字符串
let spaces = spaces.len(); // 现在是数字（usize）
// 这与变异不同——我们正在创建一个新变量
}

实际示例：计数器

// C# 版本
public class Counter
{
    private int value = 0;

    public void Increment()
    {
        value++;  // 变异
    }

    public int GetValue() => value;
}

#![allow(unused)]
fn main() {
// Rust 版本
pub struct Counter {
    value: i32,  // 默认私有
}

impl Counter {
    pub fn new() -> Counter {
        Counter { value: 0 }
    }

    pub fn increment(&mut self) {  // &mut 变异所需
        self.value += 1;
    }

    pub fn get_value(&self) -> i32 {
        self.value
    }
}
}

字符串类型：String vs &str

这是 C# 开发者最困惑的概念之一，所以让我们仔细分解。

C# 字符串处理

// C# - 简单字符串模型
string name = "John";           // 字符串字面量
string greeting = "Hello, " + name;  // 字符串连接
string upper = name.ToUpper();  // 方法调用

Rust 字符串类型

#![allow(unused)]
fn main() {
// Rust - 两种主要字符串类型

// 1. &str（字符串切片）- 类似 C# 中的 ReadOnlySpan<char>
let name: &str = "John";        // 字符串字面量（不可变，借用）

// 2. String - 类似 StringBuilder 或可变字符串
let mut greeting = String::new();       // 空字符串
greeting.push_str("Hello, ");          // 追加
greeting.push_str(name);               // 追加

// 或直接创建
let greeting = String::from("Hello, John");
let greeting = "Hello, John".to_string();  // 将 &str 转换为 String
}

何时使用哪个？

场景使用C# 等价物
字符串字面量&strstring 字面量
函数参数（只读）&strstring 或 ReadOnlySpan<char>
owned、可变字符串StringStringBuilder

实际示例

// 接受任何字符串类型的函数
fn greet(name: &str) {  // 接受 String 和 &str
    println!("Hello, {}!", name);
}

fn main() {
    let literal = "John";                    // &str
    let owned = String::from("Jane");        // String

    greet(literal);                          // 可以
    greet(&owned);                           // 可以（将 String 借用为 &str）
    greet("Bob");                            // 可以
}

// 返回_owned_字符串的函数
fn create_greeting(name: &str) -> String {
    format!("Hello, {}!", name)  // format! 宏返回 String
}

C# 开发者：这样想

#![allow(unused)]
fn main() {
// &str 类似 ReadOnlySpan<char> —— 字符串数据的视图
// String 类似你拥有并可以修改的 char[]

let borrowed: &str = "我不拥有这个数据";
let owned: String = String::from("我拥有这个数据");

// 在它们之间转换
let owned_copy: String = borrowed.to_string();  // 复制为_owned_
let borrowed_view: &str = &owned;               // 从_owned_借用
}

打印和字符串格式化

C# 开发者严重依赖 Console.WriteLine 和字符串插值（$""）。Rust 的格式化系统同样强大，但使用宏和格式说明符而不是。

基本输出

// C# 输出
Console.Write("no newline");
Console.WriteLine("with newline");
Console.Error.WriteLine("to stderr");

// 字符串插值（C# 6+）
string name = "Alice";
int age = 30;
Console.WriteLine($"{name} is {age} years old");

#![allow(unused)]
fn main() {
// Rust 输出——全是宏（注意！）
print!("no newline");              // → stdout，无换行
println!("with newline");           // → stdout + 换行
eprint!("to stderr");              // → stderr，无换行
eprintln!("to stderr with newline"); // → stderr + 换行

// 字符串格式化（类似 $"" 插值）
let name = "Alice";
let age = 30;
println!("{name} is {age} years old");     // 内联变量捕获（Rust 1.58+）
println!("{} is {} years old", name, age); // 位置参数

// format! 返回 String 而不是打印
let msg = format!("{name} is {age} years old");
}

格式说明符

// C# 格式说明符
Console.WriteLine($"{price:F2}");         // 固定小数：29.99
Console.WriteLine($"{count:D5}");         // 填充整数：00042
Console.WriteLine($"{value,10}");         // 右对齐，宽度 10
Console.WriteLine($"{value,-10}");        // 左对齐，宽度 10
Console.WriteLine($"{hex:X}");            // 十六进制：FF
Console.WriteLine($"{ratio:P1}");         // 百分比：85.0%

#![allow(unused)]
fn main() {
// Rust 格式说明符
println!("{price:.2}");          // 2 位小数：29.99
println!("{count:05}");          // 零填充，宽度 5：00042
println!("{value:>10}");         // 右对齐，宽度 10
println!("{value:<10}");         // 左对齐，宽度 10
println!("{value:^10}");         // 居中对齐，宽度 10
println!("{hex:#X}");            // 带前缀的十六进制：0xFF
println!("{hex:08X}");           // 十六进制零填充：000000FF
println!("{bits:#010b}");        // 带前缀的二进制：0b00001010
println!("{big}", big = 1_000_000); // 命名参数
}

Debug vs Display 打印

#![allow(unused)]
fn main() {
// {:?}  — Debug trait（用于开发者，自动派生）
// {:#?} — 漂亮打印的 Debug（缩进，多行）
// {}    — Display trait（用于用户，必须手动实现）

#[derive(Debug)] // 自动生成 Debug 输出
struct Point { x: f64, y: f64 }

let p = Point { x: 1.5, y: 2.7 };

println!("{:?}", p);   // Point { x: 1.5, y: 2.7 }   — 紧凑 debug
println!("{:#?}", p);  // Point {                     — 漂亮 debug
                        //     x: 1.5,
                        //     y: 2.7,
                        // }
// println!("{}", p);  // ❌ 错误：Point 没有实现 Display

// 实现 Display 用于面向用户的输出：
use std::fmt;

impl fmt::Display for Point {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        write!(f, "({}, {})", self.x, self.y)
    }
}
println!("{}", p);    // (1.5, 2.7)  — 用户友好
}
// C# 等价物：
// {:?}  ≈ object.GetType().ToString() 或反射转储
// {}    ≈ object.ToString()
// 在 C# 中你重写 ToString()；在 Rust 中你实现 Display

类型转换和转换

C# 有隐式转换、显式强制转换 (int)x 和 Convert.To*()。Rust 更严格——没有隐式数值转换。

数值转换

// C# —— 隐式和显式转换
int small = 42;
long big = small;              // 隐式扩展：好
double d = small;              // 隐式扩展：好
int truncated = (int)3.14;     // 显式缩小：3
byte b = (byte)300;            // 静默溢出：44

// 安全转换
if (int.TryParse("42", out int parsed)) { /* ... */ }

#![allow(unused)]
fn main() {
// Rust —— 所有数值转换都是显式的
let small: i32 = 42;
let big: i64 = small as i64;       // 扩展：显式使用 'as'
let d: f64 = small as f64;         // 整数到浮点数：显式
let truncated: i32 = 3.14_f64 as i32; // 缩小：3（截断）
let b: u8 = 300_u16 as u8;        // 溢出：环绕到 44（类似 C# unchecked）

// 使用 TryFrom 进行安全转换
use std::convert::TryFrom;
let safe: Result<u8, _> = u8::try_from(300_u16); // Err —— 超出范围
let ok: Result<u8, _>   = u8::try_from(42_u16);  // Ok(42)

// 字符串解析——返回 Result，不是 bool + out 参数
let parsed: Result<i32, _> = "42".parse::<i32>();   // Ok(42)
let bad: Result<i32, _>    = "abc".parse::<i32>();  // Err(ParseIntError)

// 使用 turbofish 语法：
let n = "42".parse::<f64>().unwrap(); // 42.0
}

字符串转换

// C#
int n = 42;
string s = n.ToString();          // "42"
string formatted = $"{n:X}";
int back = int.Parse(s);          // 42 或抛出
bool ok = int.TryParse(s, out int result);

#![allow(unused)]
fn main() {
// Rust —— 通过 Display 的 to_string()，通过 FromStr 的 parse()
let n: i32 = 42;
let s: String = n.to_string();            // "42"（使用 Display trait）
let formatted = format!("{n:X}");         // "2A"
let back: i32 = s.parse().unwrap();       // 42 或 panic
let result: Result<i32, _> = s.parse();   // Ok(42) —— 安全版本

// &str ↔ String 转换（Rust 中最常见的转换）
let owned: String = "hello".to_string();    // &str → String
let owned2: String = String::from("hello"); // &str → String（等价）
let borrowed: &str = &owned;               // String → &str（免费，只是借用）
}

引用转换（没有继承强制转换！）

// C# —— 向上转换和向下转换
Animal a = new Dog();              // 向上转换（隐式）
Dog d = (Dog)a;                    // 向下转换（显式，可能抛出）
if (a is Dog dog) { /* ... */ }    // 带模式匹配的安全向下转换

#![allow(unused)]
fn main() {
// Rust —— 没有继承，没有向上转换/向下转换
// 使用 trait 对象进行多态：
let animal: Box<dyn Animal> = Box::new(Dog);

// "向下转换" 需要 Any trait（很少需要）：
use std::any::Any;
if let Some(dog) = animal_any.downcast_ref::<Dog>() {
    // 使用 dog
}
// 实际上，使用枚举而不是向下转换：
enum Animal {
    Dog(Dog),
    Cat(Cat),
}
match animal {
    Animal::Dog(d) => { /* 使用 d */ }
    Animal::Cat(c) => { /* 使用 c */ }
}
}

练习

🏋️ 练习：类型安全的温度（点击展开）

创建一个 Rust 程序：

  1. 声明一个 const 表示绝对零度（摄氏度）（-273.15）
  2. 声明一个 static 计数器用于记录执行了多少次转换（使用 AtomicU32）
  3. 编写一个函数 celsius_to_fahrenheit(c: f64) -> f64，通过返回 f64::NAN 拒绝低于绝对零度的温度
  4. 通过将字符串 "98.6" 解析为 f64 然后转换来演示遮蔽
🔑 解决方案 
use std::sync::atomic::{AtomicU32, Ordering};

const ABSOLUTE_ZERO_C: f64 = -273.15;
static CONVERSION_COUNT: AtomicU32 = AtomicU32::new(0);

fn celsius_to_fahrenheit(c: f64) -> f64 {
    if c < ABSOLUTE_ZERO_C {
        return f64::NAN;
    }
    CONVERSION_COUNT.fetch_add(1, Ordering::Relaxed);
    c * 9.0 / 5.0 + 32.0
}

fn main() {
    let temp = "98.6";           // &str
    let temp: f64 = temp.parse().unwrap(); // 遮蔽为 f64
    let temp = celsius_to_fahrenheit(temp); // 遮蔽为华氏度
    println!("{temp:.1}°F");
    println!("Conversions: {}", CONVERSION_COUNT.load(Ordering::Relaxed));
}
