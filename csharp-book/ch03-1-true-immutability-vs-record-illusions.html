Rust for C# Programmers

真正的不可变性与 Record 幻觉

你将学到： 为什么 C# record 类型不是真正不可变的（可变字段、反射绕过）， Rust 如何在编译时强制执行真正的不可变性，以及何时使用内部可变性模式。

难度： 🟡 中级

C# Records —— 不可变性剧场

// C# records 看起来不可变但有逃生舱口
public record Person(string Name, int Age, List<string> Hobbies);

var person = new Person("John", 30, new List<string> { "reading" });

// 这些看起来都像是创建了新实例：
var older = person with { Age = 31 };  // 新 record
var renamed = person with { Name = "Jonathan" };  // 新 record

// 但引用类型仍然是可变的！
person.Hobbies.Add("gaming");  // 变异了原始数据！
Console.WriteLine(older.Hobbies.Count);  // 2 - older person 被影响了！
Console.WriteLine(renamed.Hobbies.Count); // 2 - renamed person 也被影响了！

// Init-only 属性仍然可以通过反射设置
typeof(Person).GetProperty("Age")?.SetValue(person, 25);

// 集合表达式有帮助但不能解决根本问题
public record BetterPerson(string Name, int Age, IReadOnlyList<string> Hobbies);

var betterPerson = new BetterPerson("Jane", 25, new List<string> { "painting" });
// 仍然可以通过转换变异：
((List<string>)betterPerson.Hobbies).Add("hacking the system");

// 甚至"不可变"集合也不是真正不可变的
using System.Collections.Immutable;
public record SafePerson(string Name, int Age, ImmutableList<string> Hobbies);
// 这更好，但需要纪律且有性能开销

Rust —— 默认真正的不可变性

#![allow(unused)]
fn main() {
#[derive(Debug, Clone)]
struct Person {
    name: String,
    age: u32,
    hobbies: Vec<String>,
}

let person = Person {
    name: "John".to_string(),
    age: 30,
    hobbies: vec!["reading".to_string()],
};

// 这根本不会编译：
// person.age = 31;  // 错误：不能赋值给不可变字段
// person.hobbies.push("gaming".to_string());  // 错误：无法可变借用

// 要修改，你必须显式选择加入 'mut'：
let mut older_person = person.clone();
older_person.age = 31;  // 现在很明显这是变异

// 或使用函数式更新模式：
let renamed = Person {
    name: "Jonathan".to_string(),
    ..person  // 从 person 复制其他字段（移动语义适用）
};

// 原始数据保证不变（直到被移动）：
println!("{:?}", person.hobbies);  // 总是 ["reading"] - 不可变

// 使用高效的不可变数据结构进行结构共享
use std::rc::Rc;

#[derive(Debug, Clone)]
struct EfficientPerson {
    name: String,
    age: u32,
    hobbies: Rc<Vec<String>>,  // 共享、不可变引用
}

// 创建新版本高效地共享数据
let person1 = EfficientPerson {
    name: "Alice".to_string(),
    age: 30,
    hobbies: Rc::new(vec!["reading".to_string(), "cycling".to_string()]),
};

let person2 = EfficientPerson {
    name: "Bob".to_string(),
    age: 25,
    hobbies: Rc::clone(&person1.hobbies),  // 共享引用，无深拷贝
};
}

练习

🏋️ 练习：证明不可变性（点击展开）

一位 C# 同事声称他们的 record 是不可变的。将这段 C# 代码翻译成 Rust 并解释为什么 Rust 版本是真正不可变的：

public record Config(string Host, int Port, List<string> AllowedOrigins);

var config = new Config("localhost", 8080, new List<string> { "example.com" });
// "不可变" record... 但是：
config.AllowedOrigins.Add("evil.com"); // 编译！List 是可变的。
  1. 创建一个真正不可变的等价 Rust 结构体
  2. 展示尝试变异 allowed_origins 是编译错误
  3. 编写一个函数创建修改后的副本（新主机）而不变异
🔑 解决方案 
#[derive(Debug, Clone)]
struct Config {
    host: String,
    port: u16,
    allowed_origins: Vec<String>,
}

impl Config {
    fn with_host(&self, host: impl Into<String>) -> Self {
        Config {
            host: host.into(),
            ..self.clone()
        }
    }
}

fn main() {
    let config = Config {
        host: "localhost".into(),
        port: 8080,
        allowed_origins: vec!["example.com".into()],
    };

    // config.allowed_origins.push("evil.com".into());
    // ❌ 错误：无法可变借用 `config.allowed_origins`

    let production = config.with_host("prod.example.com");
    println!("Dev: {:?}", config);       // 原始数据不变
    println!("Prod: {:?}", production);  // 新副本有不同的主机
}

关键见解：在 Rust 中，let config = ...（没有 mut）使整个值树不可变——包括嵌套的 Vec。C# records 仅使引用不可变，而不是内容。
