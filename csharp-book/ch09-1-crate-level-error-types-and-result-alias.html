Crate 级错误类型和 Result 别名

你将学到： 定义每个 crate 的错误枚举的生产模式与 thiserror， 创建 Result<T> 类型别名，以及何时选择 thiserror（库）vs anyhow（应用程序）。

难度： 🟡 中级

生产 Rust 的关键模式：定义每个 crate 的错误枚举和 Result 类型别名以消除样板代码。

模式

#![allow(unused)]
fn main() {
// src/error.rs
use thiserror::Error;

#[derive(Error, Debug)]
pub enum AppError {
    #[error("Database error: {0}")]
    Database(#[from] sqlx::Error),

    #[error("HTTP error: {0}")]
    Http(#[from] reqwest::Error),

    #[error("Serialization error: {0}")]
    Serialization(#[from] serde_json::Error),

    #[error("Validation error: {message}")]
    Validation { message: String },

    #[error("Not found: {entity} with id {id}")]
    NotFound { entity: String, id: String },
}

/// Crate 范围的 Result 别名——每个函数返回这个
pub type Result<T> = std::result::Result<T, AppError>;
}

在整个 Crate 中使用

#![allow(unused)]
fn main() {
use crate::error::{AppError, Result};

pub async fn get_user(id: Uuid) -> Result<User> {
    let user = sqlx::query_as!(User, "SELECT * FROM users WHERE id = $1", id)
        .fetch_optional(&pool)
        .await?;  // sqlx::Error → AppError::Database 通过 #[from]

    user.ok_or_else(|| AppError::NotFound {
        entity: "User".into(),
        id: id.to_string(),
    })
}

pub async fn create_user(req: CreateUserRequest) -> Result<User> {
    if req.name.trim().is_empty() {
        return Err(AppError::Validation {
            message: "Name cannot be empty".into(),
        });
    }
    // ...
}
}

C# 比较

// C# 等价模式
public class AppException : Exception
{
    public string ErrorCode { get; }
    public AppException(string code, string message) : base(message)
    {
        ErrorCode = code;
    }
}

// 但在 C# 中，调用者不知道会抛出什么异常！
// 在 Rust 中，错误类型在函数签名中。

为什么这重要

  • thiserror 自动生成 Display 和 Error impls
  • #[from] 启用 ? 运算符自动转换库错误
  • Result<T> 别名意味着每个函数签名干净：fn foo() -> Result<Bar>
  • 与 C# 异常不同，调用者在类型中看到所有可能的错误变体

thiserror vs anyhow：何时使用哪个

两个 crate 主导 Rust 错误处理。在它们之间选择是你要做的第一个决定：

thiserroranyhow
用途为库定义结构化错误类型为应用程序快速错误处理
输出你控制的自定义枚举不透明 anyhow::Error 包装器
调用者看到类型中的所有错误变体只是 anyhow::Error —— 不透明
最适合库 crate、API、任何有消费者的代码二进制文件、脚本、原型、CLI 工具
向下转换直接在变体上 matcherror.downcast_ref::<MyError>()
#![allow(unused)]
fn main() {
// thiserror —— 用于库（调用者需要 match 错误变体）
use thiserror::Error;

#[derive(Error, Debug)]
pub enum StorageError {
    #[error("File not found: {path}")]
    NotFound { path: String },

    #[error("Permission denied: {0}")]
    PermissionDenied(String),

    #[error(transparent)]
    Io(#[from] std::io::Error),
}

pub fn read_config(path: &str) -> Result<String, StorageError> {
    std::fs::read_to_string(path).map_err(|e| match e.kind() {
        std::io::ErrorKind::NotFound => StorageError::NotFound { path: path.into() },
        std::io::ErrorKind::PermissionDenied => StorageError::PermissionDenied(path.into()),
        _ => StorageError::Io(e),
    })
}
}
// anyhow —— 用于应用程序（只传播错误，不定义类型）
use anyhow::{Context, Result};

fn main() -> Result<()> {
    let config = std::fs::read_to_string("config.toml")
        .context("Failed to read config file")?;

    let port: u16 = config.parse()
        .context("Failed to parse port number")?;

    println!("Listening on port {port}");
    Ok(())
}
// anyhow::Result<T> = Result<T, anyhow::Error>
// .context() 为任何错误添加人类可读的上下文
// C# 比较：
// thiserror ≈ 定义带特定属性的自定义异常类
// anyhow ≈ 捕获 Exception 并用消息包装：
//   throw new InvalidOperationException("Failed to read config", ex);

指南：如果你的代码是库（其他代码调用它），使用 thiserror。如果你的代码是应用程序（最终二进制文件），使用 anyhow。许多项目两者都用——thiserror 用于库 crate 的公共 API，anyhow 用于 main() 二进制文件。

错误恢复模式

C# 开发者习惯于 try/catch 块从特定异常恢复。Rust 对 Result 使用组合器用于相同目的：

#![allow(unused)]
fn main() {
use std::fs;

// 模式 1：用回退值恢复
let config = fs::read_to_string("config.toml")
    .unwrap_or_else(|_| String::from("port = 8080"));  // 如果缺失则默认

// 模式 2：从特定错误恢复，传播其他错误
fn read_or_create(path: &str) -> Result<String, std::io::Error> {
    match fs::read_to_string(path) {
        Ok(content) => Ok(content),
        Err(e) if e.kind() == std::io::ErrorKind::NotFound => {
            let default = String::from("# new file");
            fs::write(path, &default)?;
            Ok(default)
        }
        Err(e) => Err(e),  // 传播权限错误等
    }
}

// 模式 3：传播前添加上下文
use anyhow::Context;

fn load_config() -> anyhow::Result<Config> {
    let text = fs::read_to_string("config.toml")
        .context("Failed to read config.toml")?;
    let config: Config = toml::from_str(&text)
        .context("Failed to parse config.toml")?;
    Ok(config)
}

// 模式 4：将错误映射到你的域类型
fn parse_port(s: &str) -> Result<u16, AppError> {
    s.parse::<u16>()
        .map_err(|_| AppError::Validation {
            message: format!("Invalid port: {s}"),
        })
}
}
// C# 等价物：
try { config = File.ReadAllText("config.toml"); }
catch (FileNotFoundException) { config = "port = 8080"; }  // 模式 1

try { /* ... */ }
catch (FileNotFoundException) { /* create file */ }        // 模式 2
catch { throw; }                                            // 重新抛出其他

何时恢复 vs 传播：

  • 恢复当错误有 sensible 默认或重试策略时
  • 使用 ? 传播当调用者应该决定做什么时
  • 添加上下文（.context()）在模块边界以构建错误轨迹

练习

🏋️ 练习：设计 Crate 错误类型（点击展开）

你正在构建用户注册服务。使用 thiserror 设计错误类型：

  1. 定义 RegistrationError 带变体：DuplicateEmail(String)、WeakPassword(String)、DatabaseError(#[from] sqlx::Error)、RateLimited { retry_after_secs: u64 }
  2. 创建 type Result<T> = std::result::Result<T, RegistrationError>; 别名
  3. 编写 register_user(email: &str, password: &str) -> Result<()> 演示 ? 传播和显式错误构造
🔑 解决方案 
#![allow(unused)]
fn main() {
use thiserror::Error;

#[derive(Error, Debug)]
pub enum RegistrationError {
    #[error("Email already registered: {0}")]
    DuplicateEmail(String),

    #[error("Password too weak: {0}")]
    WeakPassword(String),

    #[error("Database error")]
    Database(#[from] sqlx::Error),

    #[error("Rate limited — retry after {retry_after_secs}s")]
    RateLimited { retry_after_secs: u64 },
}

pub type Result<T> = std::result::Result<T, RegistrationError>;

pub fn register_user(email: &str, password: &str) -> Result<()> {
    if password.len() < 8 {
        return Err(RegistrationError::WeakPassword(
            "must be at least 8 characters".into(),
        ));
    }

    // 这个 ? 自动将 sqlx::Error 转换为 RegistrationError::Database
    // db.check_email_unique(email).await?;

    // 这是域逻辑的显式构造
    if email.contains("+spam") {
        return Err(RegistrationError::DuplicateEmail(email.to_string()));
    }

    Ok(())
}
}

关键模式：#[from] 启用 ? 用于库错误；显式 Err(...) 用于域逻辑。Result 别名保持每个签名干净。
