Rust for C# Programmers

宏：编写代码的代码

你将学到： 为什么 Rust 需要宏（无重载、无可变参数），macro_rules! 基础， ! 后缀约定，常见 derive 宏，以及用于快速调试的 dbg!()。

难度： 🟡 中级

C# 没有直接等价于 Rust 宏的东西。理解它们为什么存在以及如何工作消除了 C# 开发者的主要困惑来源。

// C# 有使宏不必要的特性：
Console.WriteLine("Hello");           // 方法重载（1-16 个参数）
Console.WriteLine("{0}, {1}", a, b);  // 通过 params 数组的可变参数
var list = new List<int> { 1, 2, 3 }; // 集合初始化器语法

#![allow(unused)]
fn main() {
// Rust 没有函数重载、没有可变参数、没有特殊语法。
// 宏填补这些空白：
println!("Hello");                    // 宏——在编译时处理 0+ 个参数
println!("{}, {}", a, b);             // 宏——在编译时类型检查
let list = vec![1, 2, 3];            // 宏——展开为 Vec::new() + push()
}

识别宏：! 后缀

每个宏调用以 ! 结尾。如果你看到 !，它是宏，不是函数：

#![allow(unused)]
fn main() {
println!("hello");     // 宏——在编译时生成格式化字符串代码
format!("{x}");        // 宏——返回 String，编译时格式检查
vec![1, 2, 3];         // 宏——创建并填充 Vec
todo!();               // 宏——panic 并显示"not yet implemented"
dbg!(expression);      // 宏——打印文件：行 + 表达式 + 值，返回值
assert_eq!(a, b);      // 宏——如果 a ≠ b 则 panic 并显示差异
cfg!(target_os = "linux"); // 宏——编译时平台检测
}

使用 macro_rules! 编写简单宏

// 定义一个从键值对创建 HashMap 的宏
macro_rules! hashmap {
    // 模式：以逗号分隔的 key => value 对
    ( $( $key:expr => $value:expr ),* $(,)? ) => {{
        let mut map = std::collections::HashMap::new();
        $( map.insert($key, $value); )*
        map
    }};
}

fn main() {
    let scores = hashmap! {
        "Alice" => 100,
        "Bob"   => 85,
        "Carol" => 92,
    };
    println!("{scores:?}");
}

Derive 宏：自动实现 Traits

#![allow(unused)]
fn main() {
// #[derive] 是生成 trait 实现的过程宏
#[derive(Debug, Clone, PartialEq, Eq, Hash)]
struct User {
    name: String,
    age: u32,
}
// 编译器通过检查结构体字段自动生成 Debug::fmt、Clone::clone、PartialEq::eq 等。
}
// C# 等价物：无——你会手动实现 IEquatable、ICloneable 等。
// 或使用 records：public record User(string Name, int Age);
// records 自动生成 Equals、GetHashCode、ToString——类似想法！

常见 Derive 宏

Derive用途C# 等价物
Debug{:?} 格式化字符串输出ToString() 重写
Clone通过 .clone() 深拷贝ICloneable
Copy隐式按位拷贝（无需 .clone()）值类型（struct）语义
PartialEq、Eq== 比较IEquatable<T>
PartialOrd、Ord<、> 比较 + 排序IComparable<T>
Hash用于 HashMap 键的哈希GetHashCode()
Default通过 Default::default() 默认值无参数构造函数
Serialize、DeserializeJSON/TOML 等（serde）[JsonProperty] 属性

经验法则：在每个类型上从 #[derive(Debug)] 开始。需要时添加 Clone、PartialEq。为任何跨越边界（API、文件、数据库）的类型添加 Serialize, Deserialize。

过程和属性宏（认知级别）

Derive 宏是一种过程宏——在编译时运行以生成代码的代码。你会遇到另外两种形式：

属性宏——附加到项与 #[...]：

#[tokio::main]          // 将 main() 转换为异步运行时入口点
async fn main() { }

#[test]                 // 标记函数为单元测试
fn it_works() { assert_eq!(2 + 2, 4); }

#[cfg(test)]            // 仅在测试期间条件编译此模块
mod tests { /* ... */ }

函数状宏——看起来像函数调用：

#![allow(unused)]
fn main() {
// sqlx::query! 在编译时针对数据库验证你的 SQL
let users = sqlx::query!("SELECT id, name FROM users WHERE active = $1", true)
    .fetch_all(&pool)
    .await?;
}

C# 开发者的关键见解：你很少编写过程宏——它们是高级库作者工具。但你使用它们不断（#[derive(...)]、#[tokio::main]、#[test]）。将它们视为 C# 源生成器：你受益于它们而无需实现它们。

带 #[cfg] 的条件编译

Rust 的 #[cfg] 属性类似 C# 的 #if DEBUG 预处理器指令，但类型检查：

#![allow(unused)]
fn main() {
// 仅在 Linux 上编译此函数
#[cfg(target_os = "linux")]
fn platform_specific() {
    println!("Running on Linux");
}

// 仅调试断言（类似 C# Debug.Assert）
#[cfg(debug_assertions)]
fn expensive_check(data: &[u8]) {
    assert!(data.len() < 1_000_000, "data unexpectedly large");
}

// 功能标志（类似 C# #if FEATURE_X，但在 Cargo.toml 中声明）
#[cfg(feature = "json")]
pub fn to_json<T: Serialize>(val: &T) -> String {
    serde_json::to_string(val).unwrap()
}
}
// C# 等价物
#if DEBUG
    Debug.Assert(data.Length < 1_000_000);
#endif

dbg!() —— 调试的最佳朋友

#![allow(unused)]
fn main() {
fn calculate(x: i32) -> i32 {
    let intermediate = dbg!(x * 2);     // 打印：[src/main.rs:3] x * 2 = 10
    let result = dbg!(intermediate + 1); // 打印：[src/main.rs:4] intermediate + 1 = 11
    result
}
// dbg! 打印到 stderr，包括文件：行，并返回值
// 对于调试比 Console.WriteLine 有用得多！
}
🏋️ 练习：编写 min! 宏（点击展开）

挑战：编写一个 min! 宏，接受 2 个或更多参数并返回最小值。

#![allow(unused)]
fn main() {
// 应该像这样工作：
let smallest = min!(5, 3, 8, 1, 4); // → 1
let pair = min!(10, 20);             // → 10
}
🔑 解决方案 
macro_rules! min {
    // 基本情况：单个值
    ($x:expr) => ($x);
    // 递归：将第一个与其余的最小值比较
    ($x:expr, $($rest:expr),+) => {{
        let first = $x;
        let rest = min!($($rest),+);
        if first < rest { first } else { rest }
    }};
}

fn main() {
    assert_eq!(min!(5, 3, 8, 1, 4), 1);
    assert_eq!(min!(10, 20), 10);
    assert_eq!(min!(42), 42);
    println!("All assertions passed!");
}

关键要点：macro_rules! 对令牌树使用模式匹配——它像 match 但用于代码结构而不是值。
