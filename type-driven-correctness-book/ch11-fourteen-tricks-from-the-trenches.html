八个核心模式（ch02-ch09）涵盖主要的正确构造技术。本章收集较小但高价值的技巧，这些技巧在生产 Rust 代码中反复出现 —— 每个技巧都以零或接近零的努力消除特定类别的 bug。

技巧 1 —— 边界处的哨兵 → Option

硬件协议充满哨兵值：IPMI 使用 0xFF 表示"传感器不存在"，PCI 使用 0xFFFF 表示"无设备"，SMBIOS 使用 0x00 表示"未知"。如果你将这些哨兵作为普通整数携带在代码中，每个消费者必须记住检查魔法值。如果即使一个比较忘记，你就会得到幻影 255°C 读数或虚假的供应商 ID 匹配。

规则： 在第一个解析边界将哨兵转换为 Option，并仅在序列化边界转换回哨兵。

反模式（来自 pcie_tree/src/lspci.rs）

// 哨兵在内部携带 —— 每个比较必须记住
let mut current_vendor_id: u16 = 0xFFFF;
let mut current_device_id: u16 = 0xFFFF;

正确模式（来自 nic_sel/src/events.rs）

pub struct ThermalEvent {
    pub record_id: u16,
    pub temperature: Option<u8>,  // 如果传感器报告 0xFF 则为 None
}

impl ThermalEvent {
    pub fn from_raw(record_id: u16, raw_temp: u8) -> Self {
        ThermalEvent {
            record_id,
            temperature: if raw_temp != 0xFF { Some(raw_temp) } else { None },
        }
    }
}

现在每个消费者必须处理 None 情况 —— 编译器强制它：

fn is_overtemp(temp: Option<u8>, threshold: u8) -> bool {
    temp.map_or(false, |t| t > threshold)
}

技巧 2 —— 密封 Trait

第 2 章介绍了带有将每个命令绑定到其响应的关联类型的 IpmiCmd。但有一个漏洞：如果任何代码可以实现 IpmiCmd，某人可以编写 MaliciousCmd，其 parse_response 返回错误类型或 panic。密封 trait 通过使 trait 需要只有你的 crate 可以实现的私有 supertrait 来关闭此漏洞。

mod private { pub trait Sealed {} }

pub trait IpmiCmd: private::Sealed {
    type Response;
    fn net_fn(&self) -> u8;
    fn cmd_byte(&self) -> u8;
    fn payload(&self) -> Vec<u8>;
    fn parse_response(&self, raw: &[u8]) -> io::Result<Self::Response>;
}

pub struct ReadTemp { pub sensor_id: u8 }
impl private::Sealed for ReadTemp {}
impl IpmiCmd for ReadTemp {
    type Response = Celsius;
    // ... 实现 ...
}

技巧 3 —— 用于演进枚举的 #[non_exhaustive]

标记枚举 #[non_exhaustive] 强制外部 crate 匹配它时包含通配符臂。在定义 crate 内，#[non_exhaustive] 没有效果 —— 你仍然可以编写穷举匹配。

#[non_exhaustive]
#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash)]
pub enum SkuVariant {
    S1001, S2001, S2002, S2003, S3001,
}

技巧 4 —— 类型状态构建器

第 5 章展示了协议的类型状态（会话生命周期、链路训练）。相同的想法适用于构建器 —— 结构体，其 build() / finish() 仅在所有必需字段设置后可调用。

pub struct Missing;
pub struct Set<T>(T);

pub struct DerBuilder<Mnemonic, FaultClass> {
    marker: String, fault_code: u32, mnemonic: Mnemonic, fault_class: FaultClass,
}

impl DerBuilder<Set<String>, Set<String>> {
    pub fn finish(self) -> Der { /* ... */ }
}

// ✅ 编译
DerBuilder::new("ERR", 1).mnemonic("X").fault_class("Y").finish();
// ❌ 编译错误：finish() 在 DerBuilder<Set<String>, Missing> 上不存在

技巧 5 —— FromStr 作为验证边界

对于字符串输入 —— 配置文件、CLI 参数、JSON 字段 —— 类比边界是 FromStr。

impl FromStr for DiagLevel {
    type Err = String;
    fn from_str(s: &str) -> Result<Self, Self::Err> {
        match s.to_lowercase().as_str() {
            "quick" => Ok(DiagLevel::Quick),
            "standard" => Ok(DiagLevel::Standard),
            "extended" => Ok(DiagLevel::Extended),
            other => Err(format!("unknown diag level: '{other}'")),
        }
    }
}
