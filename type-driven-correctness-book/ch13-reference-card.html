组合规则

能力令牌 + 类型状态 = 授权状态转换
类型化命令 + 量纲类型 = 物理类型响应
验证边界 + 幻象类型 = 验证配置上的类型化寄存器访问
能力混入 + 类型化命令 = 总线感知类型化操作
一次性类型 + 类型状态 = 消耗即转换协议
密封 Trait + 类型化命令 = 封闭、健全命令集
哨兵 → Option + 验证边界 = 干净的一次解析管道
类型状态构建器 + 能力令牌 = 完整构造证明
FromStr + #[non_exhaustive] = 可演进、快速失败枚举解析
常量泛型大小 + 验证边界 = 大小化、验证协议缓冲区
安全 unsafe 包装器 + 幻象类型 = 类型化、安全 MMIO
异步类型状态 + 能力令牌 = 授权异步转换
会话类型 + 类型化命令 = 完全类型化请求 - 响应通道
Pin + 类型状态 = 不能移动的自引用状态机

#[must_use] + 一次性类型 = 难以忽略、难以重用的令牌

避免的反模式

反模式为什么错误正确替代
fn read_sensor() -> f64无单位 —— 可能是 °C、°F 或 RPMfn read_sensor() -> Celsius
fn encrypt(nonce: &[u8; 12])Nonce 可以重用（借用）fn encrypt(nonce: Nonce)（移动）
fn admin_op(is_admin: bool)调用者可以说谎（true）fn admin_op(_: &AdminToken)
fn send(session: &Session)无状态保证fn send(session: &Session<Active>)
fn process(data: &[u8])未验证fn process(data: &ValidFru)
Clone 在临时密钥上击败一次性保证不要派生 Clone
let vendor_id: u16 = 0xFFFF哨兵在内部携带let vendor_id: Option<u16> = None
fn route(level: &str) 带回退拼写错误静默默认let level: DiagLevel = s.parse()?
Builder::new().finish() 无字段构造不完整对象类型状态构建器：finish() 门控在 Set 上
let buf: Vec<u8> 用于固定大小 HW 缓冲区大小仅在运行时检查RegisterBank<4096>（常量泛型）
分散的原始 unsafe { ptr::read(...) }UB 风险，未审计MmioRegion::read_u32() 安全包装器
async fn transition(&mut self)可变借用不强制执行状态async fn transition(self) -> NextState
fn cleanup() 手动调用早返回/panic 时忘记impl Drop —— 编译器插入调用
fn op() -> Result<T, String>不透明错误，无变体匹配fn op() -> Result<T, DiagError> 枚举

类型作为保证 —— 快速映射

保证Rust 等价物示例
"此证明存在"类型AdminToken
"我有证明"该类型的值let tok = authenticate()?;
"A 意味着 B"函数 fn(A) -> Bfn activate(AdminToken) -> Session<Active>
"A 和 B 两者"元组 (A, B) 或多参数fn op(a: &AdminToken, b: &LinkTrained)
"A 或 B"enum { A(A), B(B) } 或 Result<A, B>Result<Session<Active>, Error>
"总是真"()（单位类型）总是可构造
"不可能"!（从不类型）或 enum Void {}永远不能构造
