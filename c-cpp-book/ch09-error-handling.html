Rust for C/C++ Programmers

连接枚举到 Option 和 Result

你将学到： Rust 如何用 Option<T> 替换空指针，用 Result<T, E> 替换异常，以及 ? 操作符如何使错误传播简洁。这是 Rust 最独特的模式——错误是值，而不是隐藏的控制流。

  • 还记得我们之前学过的 enum 类型吗？Rust 的 Option 和 Result 只是标准库中定义的枚举：
#![allow(unused)]
fn main() {
// 这就是 Option 在 std 中的定义：
enum Option<T> {
    Some(T),  // 包含值
    None,     // 无值
}

// 以及 Result：
enum Result<T, E> {
    Ok(T),    // 成功并带有值
    Err(E),   // 错误并带有详情
}
}
  • 这意味着你学到的关于使用 match 进行模式匹配的所有内容都直接适用于 Option 和 Result
  • Rust 中没有空指针——Option<T> 是替代品，编译器强制你处理 None 情况

Rust Option 类型

  • Rust 的 Option 类型是一个只有两个变体的 enum：Some<T> 和 None
    • 这个想法是它表示一个 nullable 类型，即它要么包含该类型的有效值（Some<T>），要么没有有效值（None）
    • Option 类型用于 API 结果，操作要么成功并返回有效值，要么失败（但具体错误无关紧要）。例如，考虑将字符串解析为整数值
fn main() {
    // 返回 Option<usize>
    let a = "1234".find("1");
    match a {
        Some(a) => println!("在索引 {a} 处找到 1"),
        None => println!("找不到 1")
    }
}

Rust Option 类型

  • Rust 的 Option 可以用各种方式处理
    • unwrap() 在 Option<T> 为 None 时 panic，否则返回 T，这是最不推荐的方法
    • or() 可用于返回替代值
    • if let 让我们测试 Some<T>

生产模式：参见 使用 unwrap_or 安全地提取值 和 函数式转换：map、map_err、find_map 获取来自生产 Rust 代码的真实示例。

fn main() {
  // 这返回 Option<usize>
  let a = "1234".find("1");
  println!("{a:?} {}", a.unwrap());
  let a = "1234".find("5").or(Some(42));
  println!("{a:?}");
  if let Some(a) = "1234".find("1") {
      println!("{a}");
  } else {
    println!("字符串中未找到");
  }
  // 这将 panic
  // "1234".find("5").unwrap();
}

Rust Result 类型

  • Result 是一个类似于 Option 的 enum 类型，有两个变体：Ok<T> 或 Err<E>
    • Result 在 Rust API 中广泛使用，可能失败。想法是在成功时，函数返回 Ok<T>，否则返回特定错误 Err<T>
  use std::num::ParseIntError;
  fn main() {
  let a : Result<i32, ParseIntError>  = "1234z".parse();
  match a {
      Ok(n) => println!("解析 {n}"),
      Err(e) => println!("解析失败 {e:?}"),
  }
  let a : Result<i32, ParseIntError>  = "1234z".parse().or(Ok(-1));
  println!("{a:?}");
  if let Ok(a) = "1234".parse::<i32>() {
    println!("Let OK {a}");
  }
  // 这将 panic
  //"1234z".parse().unwrap();
}

Option 和 Result：同一枚硬币的两面

Option 和 Result 密切相关——Option<T> 本质上是 Result<T, ()>（错误不携带信息的 result）：

Option<T>Result<T, E>含义
Some(value)Ok(value)成功——值存在
NoneErr(error)失败——无值（Option）或错误详情（Result）

在它们之间转换：

fn main() {
    let opt: Option<i32> = Some(42);
    let res: Result<i32, &str> = opt.ok_or("value was None");  // Option → Result

    let res: Result<i32, &str> = Ok(42);
    let opt: Option<i32> = res.ok();  // Result → Option（丢弃错误）

    // 它们有许多相同的方法：
    // .map()、.and_then()、.unwrap_or()、.unwrap_or_else()、.is_some()/is_ok()
}

经验法则：当缺失是正常的时使用 Option（例如查找键）。当失败需要解释时使用 Result（例如文件 I/O、解析）。

练习：使用 Option 实现 log() 函数

🟢 入门

  • 实现一个 log() 函数，接受 Option<&str> 参数。如果参数为 None，它应该打印默认字符串
  • 函数应该返回 Result，成功和错误都为 ()（在这种情况下我们永远不会有错误）
解决方案（点击展开） 
fn log(message: Option<&str>) -> Result<(), ()> {
    match message {
        Some(msg) => println!("LOG: {msg}"),
        None => println!("LOG: (未提供消息)"),
    }
    Ok(())
}

fn main() {
    let _ = log(Some("系统已初始化"));
    let _ = log(None);

    // 或者使用 unwrap_or：
    let msg: Option<&str> = None;
    println!("LOG: {}", msg.unwrap_or("(默认消息)"));
}
// 输出：
// LOG: 系统已初始化
// LOG: (未提供消息)
// LOG: (默认消息)

Rust 错误处理

  • Rust 错误可以是不可恢复的（致命）或可恢复的。致命错误导致 ``panic```
    • 一般来说，应该避免导致 panics 的情况。panics 是由程序中的 bug 引起的，包括超出索引边界、在 Option<None> 上调用 unwrap() 等
    • 对于应该不可能的情况，可以有显式的 panics。panic! 或 assert! 宏可用于健全性检查
fn main() {
   let x : Option<u32> = None;
   // println!("{x}", x.unwrap()); // 将 panic
   println!("{}", x.unwrap_or(0));  // 确定——打印 0
   let x = 41;
   //assert!(x == 42); // 将 panic
   //panic!("出了问题"); // 无条件 panic
   let _a = vec![0, 1];
   // println!("{}", a[2]); // 超出边界 panic；使用 a.get(2) 将返回 Option<T>
}

// Rust 错误处理——全面且强制
use std::fs::File;
use std::io::Read;

fn read_file_content(filename: &str) -> Result<String, std::io::Error> {
    let mut file = File::open(filename)?;  // ? 自动传播错误
    let mut contents = String::new();
    file.read_to_string(&mut contents)?;
    Ok(contents)  // 成功情况
}

fn main() {
    match read_file_content("example.txt") {
        Ok(content) => println!("文件内容：{}", content),
        Err(error) => println!("读取文件失败：{}", error),
        // 编译器强制我们处理两种情况！
    }
}

Rust 错误处理

  • Rust 使用 enum Result<T, E> 枚举进行可恢复的错误处理
    • Ok<T> 变体在成功时包含结果，Err<E> 包含错误
fn main() {
    let x = "1234x".parse::<u32>();
    match x {
        Ok(x) => println!("解析数字 {x}"),
        Err(e) => println!("解析错误 {e:?}"),
    }
    let x  = "1234".parse::<u32>();
    // 与上面相同，但使用有效数字
    if let Ok(x) = &x {
        println!("解析数字 {x}")
    } else if let Err(e) = &x {
        println!("错误：{e:?}");
    }
}

Rust 错误处理

  • try 操作符 ? 是 match Ok / Err 模式的便捷简写
    • 注意方法必须返回 Result<T, E> 才能使用 ?
    • Result<T, E> 的类型可以更改。在下面的示例中，我们返回与 str::parse() 返回的相同错误类型（std::num::ParseIntError）
fn double_string_number(s : &str) -> Result<u32, std::num::ParseIntError> {
   let x = s.parse::<u32>()?; // 在错误情况下立即返回
   Ok(x*2)
}
fn main() {
    let result = double_string_number("1234");
    println!("{result:?}");
    let result = double_string_number("1234x");
    println!("{result:?}");
}

Rust 错误处理

  • 错误可以映射到其他类型，或映射到默认值（https://doc.rust-lang.org/std/result/enum.Result.html#method.unwrap_or_default）
#![allow(unused)]
fn main() {
// 在错误情况下将错误类型更改为 ()
fn double_string_number(s : &str) -> Result<u32, ()> {
   let x = s.parse::<u32>().map_err(|_|())?; // 在错误情况下立即返回
   Ok(x*2)
}
}
#![allow(unused)]
fn main() {
fn double_string_number(s : &str) -> Result<u32, ()> {
   let x = s.parse::<u32>().unwrap_or_default(); // 在解析错误情况下默认为 0
   Ok(x*2)
}
}
#![allow(unused)]
fn main() {
fn double_optional_number(x : Option<u32>) -> Result<u32, ()> {
    // ok_or 在以下情况下将 Option<None> 转换为 Result<u32, ()>
    x.ok_or(()).map(|x|x*2) // .map() 仅应用于 Ok(u32)
}
}

练习：错误处理

🟡 中级

  • 实现一个 log() 函数，带有一个 u32 参数。如果参数不是 42，返回错误。成功和错误类型的 Result<> 是 ()
  • 调用 log() 函数，如果 log() 返回错误则以相同的 Result<> 类型退出。否则打印消息说 log 已成功调用
fn log(x: u32) -> ?? {

}

fn call_log(x: u32) -> ?? {
    // 调用 log(x)，如果返回错误则立即退出
    println!("log 已成功调用");
}

fn main() {
    call_log(42);
    call_log(43);
}
解决方案（点击展开） 
fn log(x: u32) -> Result<(), ()> {
    if x == 42 {
        Ok(())
    } else {
        Err(())
    }
}

fn call_log(x: u32) -> Result<(), ()> {
    log(x)?;  // 如果 log() 返回错误则立即退出
    println!("log 已成功调用，参数为 {x}");
    Ok(())
}

fn main() {
    let _ = call_log(42);  // 打印：log 已成功调用，参数为 42
    let _ = call_log(43);  // 返回 Err(()), 无打印
}
// 输出：
// log 已成功调用，参数为 42
