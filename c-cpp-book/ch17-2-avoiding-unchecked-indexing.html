Rust for C/C++ Programmers

避免未检查的索引

你将学到： 为什么 Rust 中的 vec[i] 是危险的（越界时 panic），以及安全替代方案如 .get()、迭代器和 HashMap 的 entry() API。用显式处理替换 C++ 的未定义行为/缺失键时自动插入。

  • 在 C++ 中，vec[i] 和 map[key] 有未定义行为/缺失键时自动插入。Rust 的 [] 在越界时 panic。
  • 规则：除非你能证明索引有效，否则使用 .get() 而不是 []。

C++ → Rust 比较

// C++——静默 UB 或插入
std::vector<int> v = {1, 2, 3};
int x = v[10];        // UB！operator[] 无边界检查

std::map<std::string, int> m;
int y = m["missing"]; // 静默插入键，值为 0！

#![allow(unused)]
fn main() {
// Rust——安全替代方案
let v = vec![1, 2, 3];

// 坏：如果索引越界则 panic
// let x = v[10];

// 好：返回 Option<&i32>
let x = v.get(10);              // None——无 panic
let x = v.get(1).copied().unwrap_or(0);  // 2，或缺失时为 0
}

真实示例：使用 .and_then() 的链式安全查找

#![allow(unused)]
fn main() {
// 示例：profile.rs——双重查找：HashMap → Vec
pub fn get_processor(&self, location: &str) -> Option<&Processor> {
    self.processor_by_location
        .get(location)                              // HashMap → Option<&usize>
        .and_then(|&idx| self.processors.get(idx))   // Vec → Option<&Processor>
}
// 两个查找都返回 Option——无 panic，无 UB
}

何时 [] 可接受

  • 边界检查后：if i < v.len() { v[i] }
  • 在测试中：panic 是期望的行为
  • 使用常量：let first = v[0]; 在 assert!(!v.is_empty()); 之后

使用 unwrap_or 安全地提取值

  • unwrap() 在 None / Err 时 panic。在生产代码中，优先使用安全替代方案。

unwrap 家族

方法None/Err 时的行为何时使用
.unwrap()Panic仅测试，或证明不可失败
.expect("msg")带消息 Panic当 panic 合理时，解释为什么
.unwrap_or(default)返回 default你有便宜的常量回退
.unwrap_or_else(|| expr)调用闭包回退计算昂贵

函数式转换：map、map_err、find_map

  • Option 和 Result 上的这些方法让你转换包含的值而无需解包，用线性链替换嵌套 if/else。

快速参考

方法在作用C++ 等效
.map(|v| ...)Option / Result转换 Some/Ok 值if (opt) { *opt = transform(*opt); }
.map_err(|e| ...)Result转换 Err 值在 catch 块中添加上下文
.and_then(|v| ...)Option / Result链接返回 Option/Result 的操作嵌套 if 检查
.find_map(|v| ...)Iteratorfind + map 一次完成带 if + break 的循环
.filter(|v| ...)Option / Iterator仅保留匹配谓词的值if (!predicate) return nullopt;
.ok()?Result转换 Result → Option 并传播 Noneif (result.has_error()) return nullopt;

真实示例：用于 JSON 字段提取的 .and_then() 链

#![allow(unused)]
fn main() {
// 示例：framework.rs——查找带回退的序列号
let sys_info = json.get("SystemInfo")?;

// 首先尝试 BaseboardFru.BoardSerialNumber
if let Some(serial) = sys_info
    .get("BaseboardFru")
    .and_then(|b| b.get("BoardSerialNumber"))
    .and_then(|v| v.as_str())
    .filter(valid_serial)     // 仅接受非空、有效序列号
{
    return Some(serial.to_string());
}

// 回退到 BoardFru.SerialNumber
sys_info
    .get("BoardFru")
    .and_then(|b| b.get("SerialNumber"))
    .and_then(|v| v.as_str())
    .filter(valid_serial)
    .map(|s| s.to_string())   // 仅当 Some 时转换 &str → String
}

在 C++ 中这将是 if (json.contains("BaseboardFru")) { if (json["BaseboardFru"].contains("BoardSerialNumber")) { ... } } 的金字塔。

真实示例：find_map——搜索 + 转换一次完成

#![allow(unused)]
fn main() {
// 示例：context.rs——查找匹配传感器 + 所有者的 SDR 记录
pub fn find_for_event(&self, sensor_number: u8, owner_id: u8) -> Option<&SdrRecord> {
    self.by_sensor.get(&sensor_number).and_then(|indices| {
        indices.iter().find_map(|&i| {
            let record = &self.records[i];
            if record.sensor_owner_id() == Some(owner_id) {
                Some(record)
            } else {
                None
            }
        })
    })
}
}

find_map 是 find + map 融合：它在第一个匹配处停止并转换它。C++ 等效是带 if + break 的 for 循环。

JSON 处理：nlohmann::json → serde

  • C++ 团队通常使用 nlohmann::json 进行 JSON 解析。Rust 使用 serde + serde_json——它更强大，因为 JSON 模式编码在类型系统中。

C++ (nlohmann) vs Rust (serde) 比较

// 带 nlohmann::json 的 C++——运行时字段访问
#include <nlohmann/json.hpp>
using json = nlohmann::json;

struct Fan {
    std::string logical_id;
    std::vector<std::string> sensor_ids;
};

Fan parse_fan(const json& j) {
    Fan f;
    f.logical_id = j.at("LogicalID").get<std::string>();    // 缺失时抛出
    if (j.contains("SDRSensorIdHexes")) {                   // 手动默认处理
        f.sensor_ids = j["SDRSensorIdHexes"].get<std::vector<std::string>>();
    }
    return f;
}

#![allow(unused)]
fn main() {
// 带 serde 的 Rust——编译时模式，自动字段映射
use serde::{Serialize, Deserialize};

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct Fan {
    pub logical_id: String,
    #[serde(rename = "SDRSensorIdHexes", default)]  // JSON 键 → Rust 字段
    pub sensor_ids: Vec<String>,                     // 缺失 → 空 Vec
    #[serde(default)]
    pub sensor_names: Vec<String>,                   // 缺失 → 空 Vec
}

// 一行替换整个解析函数：
let fan: Fan = serde_json::from_str(json_str)?;
}

关键 serde 属性（来自生产 Rust 代码的真实示例）

属性目的C++ 等效
#[serde(default)]对缺失字段使用 Default::default()if (j.contains(key)) { ... } else { default; }
#[serde(rename = "Key")]将 JSON 键名映射到 Rust 字段名手动 j.at("Key") 访问
#[serde(flatten)]将未知键吸收到 HashMap 中for (auto& [k,v] : j.items()) { ... }
#[serde(skip)]不序列化/反序列化此字段不存储在 JSON 中
#[serde(tag = "type")]内部标记 enum（判别式字段）if (j["type"] == "gpu") { ... }

练习：带 serde 的 JSON 反序列化

  • 定义一个 ServerConfig 结构，可以从以下 JSON 反序列化：
{
    "hostname": "diag-node-01",
    "port": 8080,
    "debug": true,
    "modules": ["accel_diag", "nic_diag", "cpu_diag"]
}
  • 使用 #[derive(Deserialize)] 和 serde_json::from_str() 解析它
  • 为 debug 添加 #[serde(default)] 以便如果缺失则默认为 false
  • 奖励：添加带 #[serde(default)] 的 enum DiagLevel { Quick, Full, Extended } 字段，默认为 Quick

起始代码（需要 cargo add serde --features derive 和 cargo add serde_json）：

解决方案（点击展开） 
use serde::Deserialize;

#[derive(Debug, Deserialize, Default)]
enum DiagLevel {
    #[default]
    Quick,
    Full,
    Extended,
}

#[derive(Debug, Deserialize)]
struct ServerConfig {
    hostname: String,
    port: u16,
    #[serde(default)]       // 如果缺失则默认为 false
    debug: bool,
    modules: Vec<String>,
    #[serde(default)]       // 如果缺失则默认为 DiagLevel::Quick
    level: DiagLevel,
}

fn main() {
    let json_input = r#"{
        "hostname": "diag-node-01",
        "port": 8080,
        "debug": true,
        "modules": ["accel_diag", "nic_diag", "cpu_diag"]
    }"#;

    let config: ServerConfig = serde_json::from_str(json_input)
        .expect("解析 JSON 失败");
    println!("{config:#?}");

    // 测试缺失可选字段
    let minimal = r#"{
        "hostname": "node-02",
        "port": 9090,
        "modules": []
    }"#;
    let config2: ServerConfig = serde_json::from_str(minimal)
        .expect("解析最小 JSON 失败");
    println!("debug（默认）：{}", config2.debug);    // false
    println!("level（默认）：{:?}", config2.level);  // Quick
}
// 输出：
// ServerConfig {
//     hostname: "diag-node-01",
//     port: 8080,
//     debug: true,
//     modules: ["accel_diag", "nic_diag", "cpu_diag"],
//     level: Quick,
// }
// debug（默认）：false
// level（默认）：Quick
