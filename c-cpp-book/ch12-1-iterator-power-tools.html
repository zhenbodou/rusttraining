Rust for C/C++ Programmers

迭代器强大工具参考

你将学到： filter/map/collect 之外的高级迭代器组合器——enumerate、zip、chain、flat_map、scan、windows 和 chunks。对于将 C 风格索引 for 循环替换为安全、表达式的 Rust 迭代器至关重要。

基本的 filter/map/collect 链覆盖了许多情况，但 Rust 的迭代器库 要丰富得多。本节介绍你每天会用到的工具——尤其是当 翻译手动跟踪索引、累积结果或处理数据的 C 循环时 固定大小的块。

enumerate——索引 + 值（替换 C 索引循环）

fn main() {
    let sensors = ["GPU_TEMP", "CPU_TEMP", "FAN_RPM", "PSU_WATT"];

    // C 风格：for (int i = 0; i < 4; i++) printf("[%d] %s\n", i, sensors[i]);
    for (i, name) in sensors.iter().enumerate() {
        println!("[{i}] {name}");
    }

    // 查找特定传感器的索引
    let gpu_idx = sensors.iter().position(|&s| s == "GPU_TEMP");
    println!("GPU 传感器在索引：{gpu_idx:?}");  // Some(0)
}

zip——并行迭代（替换并行数组循环）

fn main() {
    let names = ["accel_diag", "nic_diag", "cpu_diag"];
    let statuses = [true, false, true];
    let durations_ms = [1200, 850, 3400];

    // C: for (int i=0; i<3; i++) printf("%s: %s (%d ms)\n", names[i], ...);
    for ((name, passed), ms) in names.iter().zip(&statuses).zip(&durations_ms) {
        let status = if *passed { "通过" } else { "失败" };
        println!("{name}: {status} ({ms} ms)");
    }
}

chain——连接迭代器

fn main() {
    let critical = vec!["ECC error", "Thermal shutdown"];
    let warnings = vec!["Link degraded", "Fan slow"];

    // 按优先级顺序处理所有事件
    let all_events: Vec<_> = critical.iter().chain(warnings.iter()).collect();
    println!("{all_events:?}");
    // ["ECC error", "Thermal shutdown", "Link degraded", "Fan slow"]
}

flat_map——展平嵌套结果

fn main() {
    let lines = vec!["gpu:42:ok", "nic:99:fail", "cpu:7:ok"];

    // 从冒号分隔的行中提取所有数值
    let numbers: Vec<u32> = lines.iter()
        .flat_map(|line| line.split(':'))
        .filter_map(|token| token.parse::<u32>().ok())
        .collect();
    println!("{numbers:?}");  // [42, 99, 7]
}

windows 和 chunks——滑动和固定大小组

fn main() {
    let temps = [65, 68, 72, 71, 75, 80, 78, 76];

    // windows(3)：大小为 3 的重叠组（如滑动平均）
    // C: for (int i = 0; i <= len-3; i++) avg(arr[i], arr[i+1], arr[i+2]);
    let moving_avg: Vec<f64> = temps.windows(3)
        .map(|w| w.iter().sum::<i32>() as f64 / 3.0)
        .collect();
    println!("移动平均：{moving_avg:.1?}");

    // chunks(2)：大小为 2 的非重叠组
    // C: for (int i = 0; i < len; i += 2) process(arr[i], arr[i+1]);
    for pair in temps.chunks(2) {
        println!("块：{pair:?}");
    }

    // chunks_exact(2)：相同，但如果有余数则 panic
    // 另外：.remainder() 给出剩余元素
}

fold 和 scan——累积

fn main() {
    let values = [10, 20, 30, 40, 50];

    // fold：单个最终结果（如 C 的累加器循环）
    let sum = values.iter().fold(0, |acc, &x| acc + x);
    println!("和：{sum}");  // 150

    // 使用 fold 构建字符串
    let csv = values.iter()
        .fold(String::new(), |acc, x| {
            if acc.is_empty() { format!("{x}") }
            else { format!("{acc},{x}") }
        });
    println!("CSV: {csv}");  // "10,20,30,40,50"

    // scan：类似 fold 但产生中间结果
    let running_sum: Vec<i32> = values.iter()
        .scan(0, |state, &x| {
            *state += x;
            Some(*state)
        })
        .collect();
    println!("运行和：{running_sum:?}");  // [10, 30, 60, 100, 150]
}

练习：传感器数据管道

给定原始传感器读数（每行一个，格式 "sensor_name:value:unit"），编写一个 迭代器管道：

  1. 将每行解析为 (name, f64, unit)
  2. 过滤低于阈值的读数
  3. 使用 fold 按传感器名称分组到 HashMap 中
  4. 打印每个传感器的平均读数
// 起始代码
fn main() {
    let raw_data = vec![
        "gpu_temp:72.5:C",
        "cpu_temp:65.0:C",
        "gpu_temp:74.2:C",
        "fan_rpm:1200.0:RPM",
        "cpu_temp:63.8:C",
        "gpu_temp:80.1:C",
        "fan_rpm:1150.0:RPM",
    ];
    let threshold = 70.0;
    // 待办：解析，过滤值 >= 阈值，按名称分组，计算平均值
}
解决方案（点击展开） 
use std::collections::HashMap;

fn main() {
    let raw_data = vec![
        "gpu_temp:72.5:C",
        "cpu_temp:65.0:C",
        "gpu_temp:74.2:C",
        "fan_rpm:1200.0:RPM",
        "cpu_temp:63.8:C",
        "gpu_temp:80.1:C",
        "fan_rpm:1150.0:RPM",
    ];
    let threshold = 70.0;

    // 解析 → 过滤 → 分组 → 平均
    let grouped = raw_data.iter()
        .filter_map(|line| {
            let parts: Vec<&str> = line.splitn(3, ':').collect();
            if parts.len() == 3 {
                let value: f64 = parts[1].parse().ok()?;
                Some((parts[0], value, parts[2]))
            } else {
                None
            }
        })
        .filter(|(_, value, _)| *value >= threshold)
        .fold(HashMap::<&str, Vec<f64>>::new(), |mut acc, (name, value, _)| {
            acc.entry(name).or_default().push(value);
            acc
        });

    for (name, values) in &grouped {
        let avg = values.iter().sum::<f64>() / values.len() as f64;
        println!("{name}: 平均={avg:.1} ({} 读数)", values.len());
    }
}
// 输出（顺序可能不同）：
// gpu_temp: 平均=75.6 (3 读数)
// fan_rpm: 平均=1175.0 (2 读数)

Rust 迭代器

  • Iterator trait 用于在用户定义的类型上实现迭代（https://doc.rust-lang.org/std/iter/trait.IntoIterator.html）
    • 在示例中，我们将为斐波那契数列实现迭代器，从 1, 1, 2, ... 开始，后继是前两个数字的和
    • Iterator 中的 associated type（type Item = u32;）定义迭代器的输出类型（u32）
    • next() 方法简单地包含实现迭代器的逻辑。在此情况下，所有状态信息都可在 Fibonacci 结构中获得
    • 我们可以实现另一个名为 IntoIterator 的 trait 来为更专门的迭代器实现 into_iter() 方法
