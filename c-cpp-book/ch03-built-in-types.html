Rust for C/C++ Programmers

内置 Rust 类型

你将学到： Rust 的基本类型（i32、u64、f64、bool、char）、类型推断、显式类型标注，以及它们与 C/C++ 原始类型的比较。没有隐式转换 — Rust 需要显式类型转换。

  • Rust 有类型推断，但也允许显式指定类型
描述类型示例
有符号整数i8, i16, i32, i64, i128, isize-1, 42, 1_00_000, 1_00_000i64
无符号整数u8, u16, u32, u64, u128, usize0, 42, 42u32, 42u64
浮点数f32, f640.0, 0.42
Unicodechar'a', '$'
布尔值booltrue, false
  • Rust 允许在数字之间任意使用 _ 以便于阅读

Rust 类型指定和赋值

  • Rust 使用 let 关键字将值赋给变量。变量的类型可以在 : 后选择性指定
fn main() {
    let x : i32 = 42;
    // 这两个赋值在逻辑上是等价的
    let y : u32 = 42;
    let z = 42u32;
}
  • 函数参数和返回值（如果有）需要显式类型。以下函数接受一个 u8 参数并返回 u32
#![allow(unused)]
fn main() {
fn foo(x : u8) -> u32
{
    return x as u32 * x as u32;
}
}
  • 未使用的变量以 _ 为前缀以避免编译器警告

Rust 类型指定和推断

  • Rust 可以根据上下文自动推断变量的类型。
  • ▶ 在 Rust Playground 中尝试
fn secret_of_life_u32(x : u32) {
    println!("The u32 secret_of_life is {}", x);
}

fn secret_of_life_u8(x : u8) {
    println!("The u8 secret_of_life is {}", x);
}

fn main() {
    let a = 42; // let 关键字赋值；a 的类型推断为 i32
    let b = 42u8; // let 关键字赋值；b 的类型推断为 u8
    secret_of_life_u32(a);
    secret_of_life_u8(b);
}

Rust 变量和可变性

  • Rust 变量默认是不可变的，除非使用 mut 关键字来声明变量是可变的。例如，以下代码除非将 let a = 42 改为 let mut a = 42，否则无法编译
fn main() {
    let a = 42; // 必须改为 let mut a = 42 才能允许下面的赋值
    a = 43;  // 除非上面改了，否则无法编译
}
  • Rust 允许重用变量名（shadowing）
fn main() {
    let a = 42;
    {
        let a = 43; // 正确：同名的不同变量
    }
    // a = 43; // 不允许
    let a = 43; // 正确：新变量和赋值
}
