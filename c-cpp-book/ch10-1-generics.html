Rust for C/C++ Programmers

Rust 泛型

你将学到： 泛型类型参数、单态化（零成本泛型）、trait 约束，以及 Rust 泛型如何与 C++ 模板比较——具有更好的错误消息且无 SFINAE。

  • 泛型允许跨数据类型重用相同的算法或数据结构
    • 泛型参数作为标识符出现在 <> 内，例如：<T>。参数可以是任何合法的标识符名称，但通常保持简短以简洁
    • 编译器在编译时执行单态化，即为遇到的每个 T 变体生成新类型
// 返回由 left 和 right（类型为 <T>）组成的 <T> 元组
fn pick<T>(x: u32, left: T, right: T) -> (T, T) {
   if x == 42 {
    (left, right)
   } else {
    (right, left)
   }
}
fn main() {
    let a = pick(42, true, false);
    let b = pick(42, "hello", "world");
    println!("{a:?}, {b:?}");
}

Rust 泛型

  • 泛型也可以应用于数据类型和相关方法。可以为特定 <T> 专门化实现（例如：f32 vs. u32）
#[derive(Debug)] // 我们稍后将讨论这个
struct Point<T> {
    x : T,
    y : T,
}
impl<T> Point<T> {
    fn new(x: T, y: T) -> Self {
        Point {x, y}
    }
    fn set_x(&mut self, x: T) {
         self.x = x;
    }
    fn set_y(&mut self, y: T) {
         self.y = y;
    }
}
impl Point<f32> {
    fn is_secret(&self) -> bool {
        self.x == 42.0
    }
}
fn main() {
    let mut p = Point::new(2, 4); // i32
    let q = Point::new(2.0, 4.0); // f32
    p.set_x(42);
    p.set_y(43);
    println!("{p:?} {q:?} {}", q.is_secret());
}

练习：泛型

🟢 入门

  • 修改 Point 类型以对 x 和 y 使用两种不同的类型（T 和 U）
解决方案（点击展开） 
#[derive(Debug)]
struct Point<T, U> {
    x: T,
    y: U,
}

impl<T, U> Point<T, U> {
    fn new(x: T, y: U) -> Self {
        Point { x, y }
    }
}

fn main() {
    let p1 = Point::new(42, 3.14);        // Point<i32, f64>
    let p2 = Point::new("hello", true);   // Point<&str, bool>
    let p3 = Point::new(1u8, 1000u64);    // Point<u8, u64>
    println!("{p1:?}");
    println!("{p2:?}");
    println!("{p3:?}");
}
// 输出：
// Point { x: 42, y: 3.14 }
// Point { x: "hello", y: true }
// Point { x: 1, y: 1000 }

组合 Rust trait 和泛型

  • Trait 可用于对泛型类型放置限制（约束）
  • 约束可以使用泛型类型参数后的 : 指定，或使用 where。以下定义了一个泛型函数 get_area，它接受任何类型 T，只要它实现 ComputeArea trait
#![allow(unused)]
fn main() {
    trait ComputeArea {
        fn area(&self) -> u64;
    }
    fn get_area<T: ComputeArea>(t: &T) -> u64 {
        t.area()
    }
}

组合 Rust trait 和泛型

  • 可以有多个 trait 约束
trait Fish {}
trait Mammal {}
struct Shark;
struct Whale;
impl Fish for Shark {}
impl Fish for Whale {}
impl Mammal for Whale {}
fn only_fish_and_mammals<T: Fish + Mammal>(_t: &T) {}
fn main() {
    let w = Whale {};
    only_fish_and_mammals(&w);
    let _s = Shark {};
    // 不会编译
    only_fish_and_mammals(&_s);
}

Rust 数据类型中的 trait 约束

  • Trait 约束可以与数据类型中的泛型组合
  • 在以下示例中，我们定义 PrintDescription trait 和一个泛型 struct Shape，其成员受 trait 约束
#![allow(unused)]
fn main() {
trait PrintDescription {
    fn print_description(&self);
}
struct Shape<S: PrintDescription> {
    shape: S,
}
// 对实现 PrintDescription 的任何类型的泛型 Shape 实现
impl<S: PrintDescription> Shape<S> {
    fn print(&self) {
        self.shape.print_description();
    }
}
}

Rust 类型状态模式和泛型

  • Rust 类型可用于在编译时强制执行状态机转换

    • 考虑一个 Drone，有两个状态：Idle 和 Flying。在 Idle 状态中，唯一允许的方法是 takeoff()。在 Flying 状态中，我们允许 land()

  • 一种方法是使用如下内容对状态机建模

#![allow(unused)]
fn main() {
enum DroneState {
    Idle,
    Flying
}
struct Drone {x: u64, y: u64, z: u64, state: DroneState}  // x, y, z 是坐标
}

Rust 类型状态模式泛型

  • 泛型允许我们在编译时强制执行状态机。这需要使用名为 PhantomData<T> 的特殊泛型
  • PhantomData<T> 是 零大小 标记数据类型。在此情况下，我们用它表示 Idle 和 Flying 状态，但它的运行时大小为 零
  • 注意 takeoff 和 land 方法采用 self 作为参数。这称为 消耗（与使用借用的 &self 对比）。基本上，一旦我们在 Drone<Idle> 上调用 takeoff()，我们只能得到 Drone<Flying>，反之亦然
#![allow(unused)]
fn main() {
struct Drone<T> {x: u64, y: u64, z: u64, state: PhantomData<T> }
impl Drone<Idle> {
    fn takeoff(self) -> Drone<Flying> {...}
}
impl Drone<Flying> {
    fn land(self) -> Drone<Idle> { ...}
}
}

Rust 类型状态模式泛型

  • 关键要点：
    • 状态可以使用 struct 表示（零大小）
    • 我们可以将状态 T 与 PhantomData<T>（零大小）组合
    • 为状态机的特定阶段实现方法现在只是 impl State<T> 的问题
    • 使用消耗 self 的方法从一个状态转换到另一个状态
    • 这给了我们 零成本 抽象。编译器可以在编译时强制执行状态机，除非状态正确，否则不可能调用方法
