Rust for C/C++ Programmers

Rust Box<T>

你将学到： Rust 的智能指针类型 — Box<T> 用于堆分配，Rc<T> 用于共享所有权，Cell<T>/RefCell<T> 用于内部可变性。这些建立在前几节的 ownership 和生命周期概念之上。你还将看到 Weak<T> 用于打破引用循环的简要介绍。

为什么使用 Box<T>？ 在 C 中，你使用 malloc/free 进行堆分配。在 C++ 中，std::unique_ptr<T> 包装 new/delete。Rust 的 Box<T> 是等价物 — 堆分配、单一所有者指针，在超出作用域时自动释放。与 malloc 不同，没有需要记住的匹配 free。与 unique_ptr 不同，没有 move 后使用 — 编译器完全阻止它。

何时使用 Box vs 栈分配：

  • 包含的类型很大，你不想在栈上复制它

  • 你需要递归类型（例如，包含自身的链表节点）

  • 你需要 trait 对象（Box<dyn Trait>）

  • Box<T> 可用于创建指向堆分配类型的指针。无论 <T> 的类型如何，指针始终是固定大小

fn main() {
    // 创建指向在堆上创建的整数（值为 42）的指针
    let f = Box::new(42);
    println!("{} {}", *f, f);
    // 克隆 box 创建新的堆分配
    let mut g = f.clone();
    *g = 43;
    println!("{f} {g}");
    // g 和 f 在这里超出作用域并自动释放
}

ownership 和 borrowing 可视化

C/C++ vs Rust：指针和所有权管理

// C - 手动内存管理，潜在问题
void c_pointer_problems() {
    int* ptr1 = malloc(sizeof(int));
    *ptr1 = 42;

    int* ptr2 = ptr1;  // 都指向同一内存
    int* ptr3 = ptr1;  // 三个指针指向同一内存

    free(ptr1);        // 释放内存

    *ptr2 = 43;        // use after free - 未定义行为！
    *ptr3 = 44;        // use after free - 未定义行为！
}

对于 C++ 开发者： 智能指针有帮助，但不能防止所有问题：

// C++ - 智能指针有帮助，但不能防止所有问题
void cpp_pointer_issues() {
    auto ptr1 = std::make_unique<int>(42);

    // auto ptr2 = ptr1;  // 编译错误：unique_ptr 不可复制
    auto ptr2 = std::move(ptr1);  // 正确：所有权转移

    // 但 C++ 仍然允许 move 后使用：
    // std::cout << *ptr1;  // 编译！但未定义行为！

    // shared_ptr 别名：
    auto shared1 = std::make_shared<int>(42);
    auto shared2 = shared1;  // 都拥有数据
    // 谁"真正"拥有它？都不是。到处都有引用计数开销。
}

#![allow(unused)]
fn main() {
// Rust - ownership 系统防止这些问题
fn rust_ownership_safety() {
    let data = Box::new(42);  // data 拥有堆分配

    let moved_data = data;    // 所有权转移给 moved_data
    // data 不再可访问 - 如果使用则编译错误

    let borrowed = &moved_data;  // 不可变借用
    println!("{}", borrowed);    // 安全使用

    // moved_data 在超出作用域时自动释放
}
}

borrowing 规则可视化

#![allow(unused)]
fn main() {
fn borrowing_rules_example() {
    let mut data = vec![1, 2, 3, 4, 5];

    // 多个不可变借用 - 可以
    let ref1 = &data;
    let ref2 = &data;
    println!("{:?} {:?}", ref1, ref2);  // 都可以使用

    // 可变借用 - 独占访问
    let ref_mut = &mut data;
    ref_mut.push(6);
    // ref_mut 活动时 ref1 和 ref2 不能使用

    // ref_mut 完成后，不可变借用再次工作
    let ref3 = &data;
    println!("{:?}", ref3);
}
}

内部可变性：Cell<T> 和 RefCell<T>

回想一下，Rust 中变量默认是不可变的。有时希望类型的大部分是只读的，同时允许对单个字段进行写访问。

#![allow(unused)]
fn main() {
struct Employee {
    employee_id : u64,   // 这必须是不可变的
    on_vacation: bool,   // 如果我们想允许对这个字段进行写访问，但使 employee_id 不可变怎么办？
}
}
  • 回想一下，Rust 允许对变量进行单个可变引用和任意数量的不可变引用 — 在编译时强制执行
  • 如果我们想传递一个员工的不可变向量，但允许更新 on_vacation 字段，同时确保 employee_id 不能被变异怎么办？

Cell<T> — Copy 类型的内部可变性

  • Cell<T> 提供内部可变性，即对否则为只读的引用的特定元素进行写访问
  • 通过复制值进出工作（.get() 需要 T: Copy）

RefCell<T> — 具有运行时 borrow 检查的内部可变性

  • RefCell<T> 提供适用于引用的变体
    • 在运行时而不是编译时强制执行 Rust borrow 检查
    • 允许单个可变借用，但如果有任何其他引用未决则panic
    • 使用 .borrow() 进行不可变访问，使用 .borrow_mut() 进行可变访问

何时选择 Cell vs RefCell

标准Cell<T>RefCell<T>
适用于Copy 类型（整数、布尔值、浮点数）任何类型（String、Vec、struct）
访问模式复制值进出（.get()、.set()）就地借用（.borrow()、.borrow_mut()）
失败模式不能失败 — 没有运行时检查如果在另一个借用活动时可变借用则panic
开销零 — 只复制字节小 — 在运行时跟踪 borrow 状态
使用时你需要不可变 struct 中的可变标志、计数器或小值你需要在不可变 struct 中变异 String、Vec 或复杂类型

共享所有权：Rc<T>

Rc<T> 允许对不可变数据进行引用计数共享所有权。如果我们想在不复制的情况下将同一个 Employee 存储在多个地方怎么办？

#[derive(Debug)]
struct Employee {
    employee_id: u64,
}
fn main() {
    let mut us_employees = vec![];
    let mut all_global_employees = Vec::<Employee>::new();
    let employee = Employee { employee_id: 42 };
    us_employees.push(employee);
    // 无法编译 — employee 已经被 move
    //all_global_employees.push(employee);
}

Rc<T> 通过允许共享不可变访问来解决问题：

  • 包含的类型自动解引用
  • 当引用计数变为 0 时类型被 drop
use std::rc::Rc;
#[derive(Debug)]
struct Employee {employee_id: u64}
fn main() {
    let mut us_employees = vec![];
    let mut all_global_employees = vec![];
    let employee = Employee { employee_id: 42 };
    let employee_rc = Rc::new(employee);
    us_employees.push(employee_rc.clone());
    all_global_employees.push(employee_rc.clone());
    let employee_one = all_global_employees.get(0); // 共享不可变引用
    for e in us_employees {
        println!("{}", e.employee_id);  // 共享不可变引用
    }
    println!("{employee_one:?}");
}

对于 C++ 开发者：智能指针映射

C++ 智能指针Rust 等价物关键区别
std::unique_ptr<T>Box<T>Rust 版本是默认值 — move 是语言级别的，不是可选的
std::shared_ptr<T>Rc<T>（单线程）/ Arc<T>（多线程）Rc 没有原子开销；仅当跨线程共享时使用 Arc
std::weak_ptr<T>Weak<T>（来自 Rc::downgrade() 或 Arc::downgrade()）相同目的：打破引用循环

关键区别：在 C++ 中，你选择使用智能指针。在 Rust 中，拥有值（T）和借用（&T）覆盖大多数用例 — 仅当你需要堆分配或共享所有权时才使用 Box/Rc/Arc。

使用 Weak<T> 打破引用循环

Rc<T> 使用引用计数 — 如果两个 Rc 值相互指向，则都不会被 drop（循环）。Weak<T> 解决这个问题：

use std::rc::{Rc, Weak};

struct Node {
    value: i32,
    parent: Option<Weak<Node>>,  // 弱引用 — 不阻止 drop
}

fn main() {
    let parent = Rc::new(Node { value: 1, parent: None });
    let child = Rc::new(Node {
        value: 2,
        parent: Some(Rc::downgrade(&parent)),  // 对 parent 的弱引用
    });

    // 要使用 Weak，尝试升级它 — 返回 Option<Rc<T>>
    if let Some(parent_rc) = child.parent.as_ref().unwrap().upgrade() {
        println!("Parent value: {}", parent_rc.value);
    }
    println!("Parent strong count: {}", Rc::strong_count(&parent)); // 1，不是 2
}

Weak<T> 在 避免过度使用 clone() 中更深入地介绍。现在，关键要点：在树/图结构中对"后向引用"使用 Weak 以避免内存泄漏。

组合 Rc 与内部可变性

当你组合 Rc<T>（共享所有权）与 Cell<T> 或 RefCell<T>（内部可变性）时，真正的力量出现了。这让多个所有者读取和修改共享数据：

模式用例
Rc<RefCell<T>>共享、可变数据（单线程）
Arc<Mutex<T>>共享、可变数据（多线程 — 参见 ch13）
Rc<Cell<T>>共享、可变 Copy 类型（简单标志、计数器）

use std::cell::{Cell, RefCell};
use std::rc::Rc;

#[derive(Debug)]
struct Employee {
    employee_id: u64,
    name: RefCell<String>,
    on_vacation: Cell<bool>,
}

fn toggle_vacation(emp: &Employee) {
    emp.on_vacation.set(!emp.on_vacation.get());
}

fn append_title(emp: &Employee, title: &str) {
    emp.name.borrow_mut().push_str(title);
}

fn main() {
    let emp = Rc::new(Employee {
        employee_id: 42,
        name: RefCell::new("Alice".to_string()),
        on_vacation: Cell::new(false),
    });

    let mut us_employees = vec![];
    let mut global_employees = vec![];
    us_employees.push(Rc::clone(&emp));
    global_employees.push(Rc::clone(&emp));

    // 通过不可变引用切换 vacation
    toggle_vacation(&emp);
    println!("On vacation: {}", emp.on_vacation.get()); // true

    // 通过不可变引用追加标题
    append_title(&emp, ", Sr. Engineer");
    println!("Name: {}", emp.name.borrow()); // "Alice, Sr. Engineer"

    // 两个 Vec 看到相同的数据（Rc 共享所有权）
    println!("US: {:?}", us_employees[0].name.borrow());
    println!("Global: {:?}", global_employees[0].name.borrow());
    println!("Rc strong count: {}", Rc::strong_count(&emp));
}
// 输出：
// On vacation: true
// Name: Alice, Sr. Engineer
// US: "Alice, Sr. Engineer"
// Global: "Alice, Sr. Engineer"
// Rc strong count: 3
