Rust for C/C++ Programmers

面向 C/C++ 程序员的 Rust 引导课程

课程概述

  • 课程概述
    • 选择 Rust 的理由（从 C 和 C++ 的角度）
    • 本地安装
    • 类型、函数、控制流、模式匹配
    • 模块、cargo
    • trait、泛型
    • 集合、错误处理
    • 闭包、内存管理、生命周期、智能指针
    • 并发
    • unsafe Rust，包括外部函数接口（FFI）
    • no_std 和嵌入式 Rust 基础（针对固件团队）
    • 案例研究：真实的 C++ 到 Rust 转换模式
  • 本课程不会涵盖 async Rust — 请参阅配套的 Async Rust Training 以全面了解 future、executor、Pin、tokio 和生产级 async 模式

自学指南

本材料既适用于讲师指导的课程，也适用于自学。如果你是独自学习，以下是如何充分利用它的建议：

进度建议：

章节主题建议时间检查点
1–4设置、类型、控制流1 天你可以编写一个 CLI 温度转换器
5–7数据结构、ownership1–2 天你可以解释为什么 let s2 = s1 会使 s1 失效
8–9模块、错误处理1 天你可以创建一个多文件项目，使用 ? 传播错误
10–12trait、泛型、闭包1–2 天你可以编写带有 trait 约束的泛型函数
13–14并发、unsafe/FFI1 天你可以使用 Arc<Mutex<T>> 编写线程安全的计数器
15–16深入探讨自行安排参考资料 — 在相关时阅读
17–19最佳实践和参考自行安排在编写实际代码时查阅

如何使用练习：

  • 每章都有动手练习，并标有难度：🟢 入门、🟡 中级、🔴 挑战
  • 在查看答案之前，务必先尝试自己完成练习。 与 borrow checker 斗争是学习过程的一部分 — 编译器的错误信息就是你的老师
  • 如果你卡住超过 15 分钟，展开答案，学习它，然后关闭答案从头再试一次
  • Rust Playground 让你无需本地安装即可运行代码

当你遇到困难时：

  • 仔细阅读编译器错误信息 — Rust 的错误提示非常有用
  • 重读相关章节；像 ownership（第 7 章）这样的概念通常在第二遍时才会豁然开朗
  • Rust 标准库文档 非常出色 — 搜索任何类型或方法
  • 对于 async 模式，请参阅配套的 Async Rust Training
